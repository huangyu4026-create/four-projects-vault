# 数字生命 GitHub 试运行说明

## 为什么先只做数字生命

数字生命资料量最大、规则最复杂，也最能检验 GitHub 是否适合承接四工程正库。

先只做数字生命，可以避免一次性迁移四个工程导致混乱。

## 第一阶段目标

1. 在 GitHub 仓库里建立 `digital-life/` 目录。
2. 放入数字生命入口、规则、文风、触发词、同步水位。
3. 不上传全量私人正文。
4. 不启用复杂自动化。
5. 不产生费用。

## 成功标准

1. 用户能在 GitHub 里看到数字生命目录。
2. Codex 能按这些文件解释数字生命工作方式。
3. 下一次数字生命同步时，能把同步水位更新到 `digital-life/04_sync/数字生命_同步水位.md`。
4. Notion 公共收件箱仍然能继续工作。
5. 本地总账仍然能判断完成状态。

## 后续扩展

如果数字生命试运行稳定，再复制同样结构到：

1. philosophy-journey
2. red-mansion
3. beauty-itself

不要在数字生命试运行未稳定前搬全量四工程。
