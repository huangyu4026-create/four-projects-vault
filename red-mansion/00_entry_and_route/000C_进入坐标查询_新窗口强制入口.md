## 一句话定案

红楼梦工程从此有两个头，后面的身子和尾巴共用。

旧头 / 聚拢头：

```text
进入聚拢查询
```

走旧语义 / 聚拢 / 闭环包路线。

新头 / 坐标头：

```text
进入坐标查询
```

或用户明确说：

```text
用坐标法查
```

第一步必须走坐标取材门。

触发词里尽量不要写：

```text

```

因为“红楼梦工程”会先触发旧 `000`，导致坐标题被旧闭环包吸走。

## 总铁律

坐标题不得先建旧闭环包。

坐标题第一动作是：

```text
取材门：坐标
recall_gate：coordinate
```

旧闭环包、聚拢库、材料池不是废掉，而是后面承接材料。

坐标头只替换“取材入口”，不替换整个工程身体。

坐标头的固定工具是：

```text
python3 work/formal_honglou_coordinate_material_pack_cli.py --question "用户问题"
```

这个工具负责：

```text

自动生成距离、共场、交集候选；
自动写出材料池入口包；
```


## 坐标头强制流程

只要触发坐标头，必须按下列顺序：

```text
0. 确认坐标头触发。
1. 写入：
   取材门：坐标
   recall_gate：coordinate
2. AI 取词：生成 Codex查询词、Codex强复合、Codex词角色、Codex材料升级条件。
3. 查询逻辑策略选择：必须生成 Codex查询逻辑策略；若无合适模板，选模板00｜拆子问题策略，并写 Codex子问题 与 Codex子问题策略队列。
4. 定执行顺序：生成 Codex查证顺序。
5. 人物 / 名字 / 称谓 / 别名题必须先过 70/72 全人物归一包。
6. 进入坐标材料门。
7. 有明确原文字词时，走全文词位材料索引库防漏。
8. 有人物、空间、季节、物象、事件时，走全量变量投影库定点。
9. 问最近 / 靠近 / 同场 / 同时 / 共同出现时，必须做距离、共场、交集。
10. 输出原子段、atom_code、segment_no、原文句、变量点、距离矩阵。
11. 送入同一个材料池入口。
12. 到材料池入口后，交还旧工程后半身。
```

## AI取词硬规则

坐标头不靠机器硬切自然句。

进入坐标查询以后，必须先由 AI 读题取词，再让坐标材料门验词、归一、查表、入材料池。

```text
自然问题
  -> AI 取词
  -> Codex查询词 / Codex强复合 / Codex词角色 / Codex材料升级条件
  -> Codex查询逻辑策略
  -> Codex查证顺序
  -> formal_honglou_coordinate_material_gate.material_pack_bundle()
  -> 材料池入口候选
```

硬规则：

```text
人物轴：人名不拆、不猜；必须先接 70/72 全人物归一包；变量投影里人物只按 variable_value_name 精确命中。
变量轴：先查 variable_value_name 精确命中；没有精确命中时，二字以上词才允许包含补救。
词位轴：明确正文词 / 短语 / 单字，先走全文词位库穷尽。
单字轴：单字分三层处理，不能一刀切。
  1. 物象根字可以保留：如竹可以引出竹子、竹叶、竹梢；金可以引出黄金、金器；花、泪、黄等可作物象/颜色/意象根轴。
  2. 明确问字词时可以保留：如“黄字”“金这个字”“眼泪的泪”。
  3. 人名拆字和宽网附带字不得自动升主查：如“黛玉”的“玉”、由“泪”宽泛带出的“珠”，只能作补查提示，不能冒充主轴。
代词轴：我、你、他、她、自己、本人不得作为主查词；它们只能作为说话边界、叙述视角或裁判条件。
同场轴：same_atom / same_scene / same_event 只表示候选接近，不等于最终结论。
原文轴：每个候选必须带 atom_code、segment_no、chapter_no、source_sentence。
```

## 继承聚拢查询取词心法

坐标门不是另起一套乱词法。

坐标门继承旧聚拢查询中已经成熟的 Codex 查询词路心法：

```text
AI 先定中心。
AI 再定路径。
AI 再定词网。
AI 再定执行顺序。
工具只按这些令牌查库，不替 AI 猜词。
```

坐标门的 AI 取词必须完成五件事：

```text
1. 定中心：本题到底是人物、物象、空间、季节、事件、原句、距离、同场，还是多轴交叉。
2. 定主查词：只放原问题必须成立的核心对象。
3. 定强复合：把必须共同成立的对象放进 compound_groups，防止单边材料冒充关系证据。
4. 定词角色：区分主查词、归一词、扩展词、背景锚点、排除词。
5. 定升级条件：什么材料可以入材料池，什么只能算候选或补查入口。
```

定完这五件事后，必须进入查询逻辑策略模板组选择执行策略。这里不是“取词策略”，而是取词之后的查询逻辑策略：

```text
000E_查询逻辑策略模板组_新窗口学习入口.md
```

也就是：先 AI 取词，再定中心、定路径、定词网，最后在查询逻辑策略模板组里定执行顺序。人物题、人物+人物题、人物+物象交集题、贡献题、忌讳题、诗词文题等，都应先落入对应查询逻辑策略模板，再生成 Codex查证顺序。

强制字段：

```text
Codex查询逻辑策略：模板编号｜模板名称｜为什么选它
```

如果所有策略都不合适，必须写：

```text
Codex查询逻辑策略：模板00｜拆子问题策略
Codex子问题：
1. ____
2. ____
Codex子问题策略队列：
1. 子问题：____｜两两可查：对象A + 对象B｜重新选择策略：模板__｜Codex查询词：____｜Codex强复合：____｜Codex查证顺序：____｜Codex材料升级条件：____
2. 子问题：____｜两两可查：对象A + 对象B｜重新选择策略：模板__｜Codex查询词：____｜Codex强复合：____｜Codex查证顺序：____｜Codex材料升级条件：____
```

模板00的子问题不能只列标题。子问题产生后必须分别排队，每个子问题重新进入 `000E` 的查询逻辑策略选择；四个子问题就是四次策略选择。每条子问题必须写成两两可查的对象或强复合，再生成自己的查证顺序。

词的身份不可混：

```text
主查词：本题真正要证明关系的核心对象，进入 Codex查询词 和 Codex强复合。
归一词：人物别名、简称、繁简、异写；交给 70/72 人物归一或正字归一，不当第一轮执行词。
扩展词：同类物象、动作、状态、场景词，只用于补漏，不能自动升主证。
背景锚点：地点、回目、标题、名场面、诗词入口，只帮助定位，不能单独冒充答案。
排除词：容易串题、同字误召回、旧题污染、标题诱导和无关热词，用于降级或补查。
```

坐标门特别继承旧路的几个判断：

```text
人物对象：人名不拆、不猜。人物组只放归一人物实体，例如林黛玉；黛玉、林姑娘、颦儿等放归一词，由人物包处理。
物象对象：可抓最小有效物象字/词。竹可以作为竹子、竹叶、竹梢、竹影等物象根；金、玉、泪、帕、灯等单字物象也可以作主查词。
复合物对象：必须守住本体。竹扇可提示扇和竹两线，但普通扇不能自动算竹，必须靠原文或变量证据升级。
关系题：第一组通常是人物/主体，第二组是物象/空间/事件/季节；动作、情绪、地点、回目多为辅助，不轻易进强复合。
补查题：首轮单边、标题化、摘要化、材料互不支撑时，回到 AI 取词层调整词角色和强复合，不让程序自动扩大。
```

## 诗词文题取材专门规则

凡用户问某回、某人、某场景有没有诗、词、曲、歌、赋、判词、对联、题咏等诗词文内容，不要先把整回全文轮扫当作第一取材动作。

第一取材入口应优先走：

```text
literary_texts_axis / 诗词文库
```

执行顺序：

```text
1. 若题目带回目，先用 chapter_no 锁回。
2. 若题目带人物，再叠加 target_people / 人物变量。
3. 若题目带场景，再叠加 segment_no / scene / space / event 相关锚点。
4. 若题目带文体，再用 text_type / title / theme_tags 筛。
5. 诗词文库取到结构化清单后，再用全文词位和变量投影补漏。
6. 最后只回 chapters.full_text 抽检关键段，不把整回通读当第一动作。
```

词位与变量交叉在这里不是第一刀，而是补漏和裁判辅助：

```text
诗词文库负责先给结构化清单。
全文词位负责防漏：诗、词、曲、歌、赋、判词、对联、题曰、其判曰、其词曰等。
变量投影负责定点：poem / prophecy / event / object / space / person 等。
chapters.full_text 负责最终原文裁判。
```

降级条件：

```text
诗词文库命中不等于最终答案。
同一文本可能被重复分类，如曲 / 红楼梦曲、判词 / 七绝 / 诗谶。
遇到异文、重复分类、标题化记录、边界不明时，必须回原文抽检。
```

坐标门与旧聚拢门的区别只在执行库：

```text
旧聚拢门：AI取词 -> 聚拢/语义/闭环包取材 -> 材料池
坐标门：AI取词 -> 词位库/变量投影库/字位桥表/距离共场 -> 材料池
```

所以，坐标门不是“更聪明地切词”，而是“更硬地执行 AI 已经想清楚的词路”。

如果模型发现自己在现场写专项脚本，说明入口工具没有用对；先人工智能取词，再改用：

```text
python3 work/formal_honglou_coordinate_material_pack_cli.py --question "用户问题" --route-context "取材门：坐标
recall_gate：coordinate
Codex查询词：...
Codex强复合：...
Codex词角色：...
Codex查证顺序：...
Codex材料升级条件：..."
```

## 对象词提醒

对象词不要被单一路径锁死。

```text
全文词位负责防漏。
变量坐标负责定点。
变量未登记不等于原文没有。
精确变量也不能替代全文词位和原文裁判。
```

例如：

```text
花字：走全文词位 term_norm='花'
花物象：走变量 object=花
花相关扩展：由 AI 放在扩展词、背景锚点或补查方案中
```

最终靠人物、空间、季节、事件、距离 / 共场交叉裁判。

## 证据层级提醒

坐标候选必须标清证据层级：

```text
evidence_layer=原文句
evidence_layer=全文词位
evidence_layer=工程变量点:person/object/space/season/event
evidence_layer=段落摘要
evidence_layer=工程推定
```

只有 `原文句` 可以直接进入原文裁判。

`工程变量点`、`段落摘要`、`工程推定` 都只能定位和组证，不能冒充原文。

坐标包必须同时输出：

```text
material_status=可入池 / 需补证
final_answer_status=不可作答，需回原文裁判
```

`可入池` 只表示候选可以交材料池四态，不表示可直接回答。

## 自述梦裁判门

遇到“某人自己说梦 / 亲口说做梦 / 醒后讲梦 / 自述梦”这类题，不新增大库，只加裁判门。

必须分四类：

```text
叙述者说明：叙述者说某人做梦。
梦中台词：人在梦里说话。
醒后自述：醒来以后本人说自己做了梦。
他人转述：别人说某人做梦或梦见什么。
```

只有“醒后自述”可以回答“本人亲口说我做了一个梦”。

第82回这种材料，坐标门可以快速定位梦境现场，但最终裁判必须回原文区分：

```text
有梦。
有梦中台词。
有醒后惊醒和反应。
不等于醒后本人亲口复述“我做了一个梦”。
```

```text
人物归一：通过 / 未触发 / 需补。
坐标预检：通过 / 需补证。
坐标取材包：package_dir=____。
坐标入材料池凭证门：准入待判 / 阻断待补。
下一步：材料池四态判定，不写最终答案。
```

短卡只报告状态，不代替材料池判定。

如果缺 `Codex查询词`，短卡只能写：

```text
坐标头阻断：缺 Codex查询词。必须由 AI 先取词，工具不裸猜。
```

如果已经形成坐标候选，但未形成 `04_坐标入材料池凭证门.md/json`，短卡只能写：

```text
坐标取材已命中，待补坐标入材料池凭证门。
```

## 坐标双打卡

旧聚拢头的 130 双打卡不照抄，但思想必须保留。

坐标头用两层打卡：

```text
第一层：坐标判题打卡。
第二层：坐标入池凭证打卡。
```

第一层只确认这题能不能进入坐标取材：

| 字段 | 通过标准 |
|---|---|
| 取材门 | 必须是坐标。 |
| 题目中心 | 已判断人物 / 物象 / 空间 / 季节 / 事件 / 原句 / 距离 / 同场 / 多轴。 |
| Codex查询词 | 已由 AI 给出，不由工具裸猜。 |
| 人物归一 | 凡人名、称谓、别名题必须先归一。 |
| 强复合 | 关系题、同场题、距离题必须有强复合轴或说明为什么不需要。 |

第二层只确认坐标候选能不能交给材料池：

| 字段 | 通过标准 |
|---|---|
| 坐标来源 | 有词位库 / 变量投影库 / 字位桥表 / 距离共场来源说明。 |
| 原子段 | 有 atom_code 或 segment_no。 |
| 原文锚点 | 有 chapter_no、source_sentence 或 material_source_sentences。 |
| 路由说明 | 有 `codex_query_lane=coordinate` 或等价坐标路由说明。 |
| 入池凭证 | 生成 `04_坐标入材料池凭证门.md/json`。 |

状态只能三种：

```text
坐标绿灯：可交材料池四态判定。
坐标黄灯：有候选但凭证不全，补锚点或补词路。
坐标红灯：缺入口词 / 缺人物归一 / 缺坐标候选，不准进材料池。
```

坐标绿灯也不是最终答案。

坐标绿灯只表示：

```text
可以交给材料池四态判定。
```

## 坐标入材料池凭证门

坐标取材完成后，必须生成：

```text
04_坐标入材料池凭证门.md
04_坐标入材料池凭证门.json
```

这两个文件对应旧路线的“入材料池凭证门”思想，但不伪装成旧 `00AJ`。

它们必须写清：

```text
material_admission_status：准入待判 / 阻断待补
material_admission_reason：为什么能交材料池，或为什么阻断
material_admission_rule：坐标候选必须具备坐标取材来源、入口词路、原子段/回目/原文句锚点
codex_query_lane：coordinate
source_system：坐标材料门
source_trace：词位库 / 变量投影库 / 字位桥表 / 距离共场
atom_code / segment_no / chapter_no / source_sentence
evidence_layer / final_answer_status
material_term_hits / material_coordinate_hits / material_nearest_pairs / material_cooccurrence
```

如果 `04_坐标入材料池凭证门` 缺失，状态不能写“已入材料池”，只能写：

```text
坐标取材已命中，待补坐标入材料池凭证门。
```

如果凭证门状态是 `阻断待补`，材料不能进入红楼解语，只能进入补证。

## 坐标头验收矩阵

坐标头成功与否，不用旧闭环包四件套验前半段。

必须用下表验收：

| 验收项 | 通过标准 |
|---|---|
| 坐标触发 | 出现 `取材门：坐标` 和 `recall_gate：coordinate`。 |
| 查询词路 | 出现 `Codex查询词`、`Codex强复合`、`Codex词角色`。 |
| 人物归一 | 人物题出现 `character_key` 和归一正名。 |
| 坐标预检 | `coordinate_preflight_status=通过`，或明确写出需补证令牌。 |
| 词位取材 | 有 `material_term_hits`，或说明本题没有明确字词入口。 |
| 变量取材 | 有 `material_coordinate_hits`，或说明本题没有变量入口。 |
| 距离 / 共场 | 最近、同场、同时、共同出现题必须有 `material_nearest_pairs` 或 `material_cooccurrence`。 |
| 原子段 | 有 `atom_code`、`segment_no`、`global_atom_order` 中的可用字段。 |
| 原文句 | 有 `material_source_sentences` 或等价原文句。 |
| 材料池入口 | 有 `04_坐标入材料池凭证门.md/json`，且状态为 `准入待判` 或明确阻断原因。 |
| 写作阻断 | 未完成材料池四态、精读材料词、写作前追证前，不写最终答案。 |

如果缺少坐标触发、查询词路、人物归一强令牌或坐标预检，状态只能写：

```text
坐标头阻断：未完成坐标取材入口。
```

如果已有坐标材料候选但尚未送入材料池入口，状态只能写：

```text
坐标取材已命中，待入材料池。
```

如果已送入材料池入口但后半身未完成，状态只能写：

```text
坐标材料已入池，待材料池四态 / 精读材料词 / 写作前追证。
```

## 状态词

坐标头只能这样报告状态：

```text
已触发坐标头 = 已识别坐标法，并生成取材门：坐标。
已进入坐标材料门 = 已通过坐标预检并调用坐标材料门。
已完成坐标取材 = 已提出原子段、原文句、变量点或词位点。
已送入材料池入口 = 坐标候选已进入总证据池 / 材料池入口。
可写最终答案 = 仍需旧工程后半身完成材料池四态、精读材料词、写作前原文追证。
```

## 一句话给新线程

如果用户说：

```text
进入红楼梦工程，用坐标法查询……
```

新线程必须理解成：

```text
进入坐标查询。
先走坐标头。
不得先走旧闭环包。
坐标取材完成后，再把材料送回同一个材料池入口。
```

更推荐用户新开页时直接说：

```text
进入坐标查询。
我要查：……
```

如果要走旧聚拢路线，则说：

```text
进入聚拢查询。
我要查：……
```
