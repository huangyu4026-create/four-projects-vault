# B轮聚拢复核提问稿_设计跑

## 复核对象

本轮不是重新回答原问题，也不是重新跑全文召回。

本轮只接收 `B0_A轮坐标结论交接包.json` 中的 `coordinate_claims`，逐条复核 A轮坐标法提出的候选结论。

## 给聚拢法的问题

请读取每条 `claim_id` 的 `atom_code / segment_no / scene_id / event_id / time_block_id / source_sentence / coordinate_reason`，只围绕这些锚点做聚拢复核：

1. 这个坐标候选是否构成真实同场？
2. 宝玉、黛玉、手帕、泪象是在同一原子段、同一场、同一事件，还是只是同时间块/同回背景？
3. 如果 source_sentence 不足以证明四轴同场，需要补读哪些前后段？
4. 这条材料应判为主证、旁证、背景、误召回，还是需补证？
5. 是否允许进入材料池四态？

## 禁止动作

- 禁止把原问题重新生成关键词再跑一遍。
- 禁止引入不在 A轮 claims 内的宽候选。
- 禁止把同回共现当同场。
- 禁止把坐标准入待判当最终答案。
- 禁止只凭摘要或工程推定定案。

## 期望产物

- `B1_聚拢复核清单_设计跑.csv` 填入每条 claim 的复核结果。
- 必要时另列 `B2_现场链补强表.csv`，只写需要补读的前后段/聚拢单元/聚拢事件。
- 通过复核的材料再进入材料池四态；未通过的降为背景、误召回或需补证。
