# 000E_查询逻辑策略模板组_新窗口学习入口

生成时间：2026-06-22

性质：查询逻辑策略模板组 / 位于 AI 取词之后 / 决定定中心、定路径、定词网之后的执行顺序

状态：候选正式头；不改旧库；不自动写 SQLite；供聚拢查询与坐标查询共同调用

## 一句话定案

AI 取词只负责提炼词位、变量、别名、扩展词、排除词。

查询逻辑策略负责决定：这些词按什么中心、路径、库表和顺序被查。

AI 取词完成后，必须进入查询逻辑策略模板组，把题目压成一个可执行的查询策略。

```text
自然问题
  -> AI取词：取词位 / 变量 / 别名 / 扩展词 / 排除词
  -> 定中心
  -> 定路径
  -> 定词网
  -> 选择查询逻辑策略模板
  -> 定执行顺序
  -> 生成 Codex查询词 / Codex强复合 / Codex词角色 / Codex查证顺序 / Codex材料升级条件
```

## 双头共用规则

本文件不是坐标头专用，也不是聚拢头专用。

它是两头共用的查询逻辑策略层：

```text
进入坐标查询 -> AI取词 -> 定中心 -> 定路径 -> 定词网 -> 选择查询逻辑策略模板 -> 坐标材料门执行
进入聚拢查询 -> AI取词 -> 定中心 -> 定路径 -> 定词网 -> 选择查询逻辑策略模板 -> 聚拢库 / 聚拢总图执行
```

两头的共同点：

```text
都先读题。
都先判断题型。
都先拆对象。
都先定主轴。
都必须选择 Codex查询逻辑策略。
复杂题都必须拆成两两可查子问题，并让每个子问题重新进入策略选择。
最终都必须回原文裁判，不能把命中当答案。
```

两头的差别只在后端执行：

```text
坐标头：编码坐标查、库表验链、原文裁判；重点是词位、变量、原子段、距离、同场、坐标材料门。
聚拢头：聚拢总图、聚拢四件套、放大/缩小/交集/串域、材料池四态；重点是编号、聚拢段、聚拢单元、聚拢事件、聚拢场。
```

因此，策略模板先于后端。

不管本题最后走坐标材料门还是聚拢库，都必须先回答：

```text
这题是什么问题形状？
主中心是什么？
要先查哪个库轴？
强复合对象是哪两个？
如果不止两个对象，怎样拆成两两队列？
每个子问题各自选择哪个策略？
```

```text
人物、关系、物象、空间、事件、诗词、时间、主题、全文穷尽，都必须先变成查询逻辑策略。
策略选定后，坐标头负责编码坐标查；聚拢头负责聚拢现场裁判。
```

## 强制选择闸门

查询逻辑策略节点必须做一次显式选择。

强制的是有策略意识，不是强制死模板执行。

策略卡是起手卡，不是死模板：

```text
卡要拿，方法不能死。
必须选择一张起手策略卡；复杂题可以组合多张卡。
证据反馈后可以换卡；无法按卡直走时可以偏离执行。
组合、换卡、偏离都必须留理由，最后仍回原文裁判。
```

```text
Codex查询逻辑策略：模板编号｜模板名称｜为什么选它
Codex策略执行口径：起手卡 / 组合卡 / 换卡 / 偏离执行
Codex策略组合：主卡=____；辅卡=____；原因=____
Codex策略调整记录：原卡=____；新卡=____；触发证据=____；为什么换
Codex偏离理由：____
```

如果现有策略模板都不合适，不允许跳过。

必须选择兜底策略：

```text
Codex查询逻辑策略：模板00｜拆子问题策略
Codex子问题：
1. ____
2. ____
3. ____
Codex子问题策略队列：
1. 子问题1｜两两可查：对象A + 对象B / 变量A + 变量B｜重新选择策略：模板__｜Codex查询词：____｜Codex强复合：____｜Codex查证顺序：____
2. 子问题2｜两两可查：对象A + 对象B / 变量A + 变量B｜重新选择策略：模板__｜Codex查询词：____｜Codex强复合：____｜Codex查证顺序：____
3. 子问题3｜两两可查：对象A + 对象B / 变量A + 变量B｜重新选择策略：模板__｜Codex查询词：____｜Codex强复合：____｜Codex查证顺序：____
```

硬阻断：

```text
没有 Codex查询逻辑策略，不进入坐标材料门。
写“待定 / 不确定 / 不用策略 / 无策略”，视为没有选择。
选择模板00但没有 Codex子问题策略队列，不进入坐标材料门。
子问题有几个，策略队列就必须有几条；四个子问题必须进入四次重新策略选择。
已有策略后，工程不要求死按卡执行；但组合、换卡、偏离必须能说明理由。
```

查询逻辑策略模板组解决的是：

```text
这题属于什么题型？
第一刀走哪个库？
哪些词是主查词？
哪些词只是归一、背景、补漏或排除？
哪些材料可以升级？
最后回哪一层原文裁判？
```

## 总铁律

查询逻辑策略模板只帮助 AI 决定查什么、先查哪里、怎么补漏。

查询逻辑策略模板不是答案模板，也不是取词策略。

```text
模板是思路起点，不是答案，也不是死流程。
模板只到候选、查证顺序和证据反馈后的调整方向。
最终结论仍必须回原文、入材料池、做四态判定。
```

## 策略模板 00｜拆子问题策略

触发题型：一个问题同时包含多个中心、多个关系、多个判断阈值，或无法落入单一策略模板。

定中心：先定总中心，再拆成若干子中心。

第一路径：先拆子问题，再让每个子问题分别选择 01-10 的具体查询逻辑策略。

```text
自然问题
-> 判断不能单模板覆盖
-> Codex查询逻辑策略：模板00｜拆子问题策略
-> Codex子问题清单
-> 子问题逐条排队
-> 每个子问题重新进入查询逻辑策略选择
-> 每个子问题分别写 Codex查询逻辑策略 / Codex查询词 / Codex强复合 / Codex查证顺序 / Codex材料升级条件
```

子问题设计硬规则：

```text
1. 每个子问题必须是两两可查的问题单元。
2. 两两可以是：人物A + 人物B，人物 + 物象，人物 + 行动，场景 + 人物，原句 + 说话人，文体 + 章回，判断命题 + 反证对象。
3. 子问题不是小标题；它必须能生成自己的 Codex查询词、Codex强复合、Codex查证顺序和材料升级条件。
4. 子问题生成以后，不沿用总问题的策略；每个子问题都重新进入本文件的模板选择顺序。
5. 四个子问题就是四条队列、四次策略选择、四套材料升级条件。
```

子问题策略队列格式：

```text
Codex子问题策略队列：
1. 子问题：____｜两两可查：对象A + 对象B｜重新选择策略：模板__｜Codex查询词：____｜Codex强复合：____｜Codex查证顺序：____｜Codex材料升级条件：____
2. 子问题：____｜两两可查：对象A + 对象B｜重新选择策略：模板__｜Codex查询词：____｜Codex强复合：____｜Codex查证顺序：____｜Codex材料升级条件：____
```

材料升级条件：每个子问题必须有自己的候选、原文锚点和材料升级条件；不能用一个子问题的证据替另一个子问题作答。

降级条件：如果只拆了题但没有给出子问题策略队列，仍视为未完成查询逻辑策略选择。

## 策略模板 01｜人物单体题

触发题型：查某个人是谁、别名、身份、行为、命运、性格、某次言行。

定中心：人物中心。

第一路径：人物归一。

```text
characters / character_alias_solidification
-> person_segment_edges
-> segments
-> chapters.full_text
```

主查词：归一人物实体名。

词角色：别名、称谓、异写只作归一词和补漏词，不直接升主查。

材料升级条件：同一人物的称谓、言行、动作和上下文能共同支撑判断。

降级条件：只命中同名、旁人谈及、摘要标题，不得升主证。

## 策略模板 02｜人物 + 人物 / 多人物关系题

触发题型：查两人或多人有没有同场、交流、关系、冲突、亲疏、互相影响。

定中心：关系中心 / 共现查证中心。

第一路径：人物共现映射。

```text
人物归一
-> person_segment_edges 多人物同段/同场候选
-> evidence_edges 关系边
-> segment_range_edges 连续场景补证
-> segments / chapters.full_text 原文裁判
```

强复合：人物 A + 人物 B；多人题必须把全部核心人物放进强复合。

材料升级条件：人物双方在同一原文语境里形成动作、对话、观察、评价、影响或明确同场关系。

降级条件：同回不同场、两两拼接、旁人转述、标题/摘要命中，只能作候选或反证。

## 策略模板 03｜人物 + 物象 / 交集题

触发题型：查某人与某物、器物、植物、信物、空间物有没有交集、接触、使用、看见、同场。

定中心：人物-物象复合关系。

第一路径：人物归一 + 物象轴。

```text
人物归一
-> objects_axis 锁物象实体和同类词
-> evidence_edges / person_segment_edges 查共现候选
-> segments / chapters.full_text 回原文裁判
```

强复合：人物实体 + 物象实体。

材料升级条件：人物与物象在同一原子段、连续场景或明确动作链中共同成立。

降级条件：物象只是背景、诗词意象、空间名、别号、标题字，不能自动算接触或使用。

## 策略模板 04｜首次 / 第一次 / 顺序题

触发题型：第一次、首次、最早、先后、变化、从什么时候开始。

定中心：时间顺序中心。

第一路径：先召回全候选，再按章回和字位排序。

```text
相关实体归一
-> 相关轴库召回候选
-> 全文词位补漏
-> 按 chapter_no / segment_no / start_char 排序
-> 逐条回 chapters.full_text 裁判阈值是否成立
```

材料升级条件：必须说明“成立阈值”：出现、同场、接触、使用、亲口说、实际行动等，不能混为一谈。

降级条件：早期背景、伏笔、意象、别人转述不自动算首次成立。

## 策略模板 05｜诗词文题

触发题型：问某回、某人、某场景有没有诗、词、曲、歌、赋、判词、对联、题咏、谜语等内容。

定中心：文学文本中心。

第一路径：诗词文库。

```text
chapter_no / 人物 / 场景 锁定范围
-> literary_texts_axis / 诗词文库
-> text_type / title / theme_tags 筛文体
-> 全文词位和变量投影补漏
-> chapters.full_text 抽检关键段
```

材料升级条件：诗词文库给出结构化清单，原文抽检确认文本边界和上下文。

降级条件：同一文本重复分类、异文、标题化记录、边界不明时，不直接作答，必须回原文。

## 策略模板 06｜原句 / 词义 / 语气解释题

触发题型：某句话是什么意思、某个词是什么意思、这句话是谁说的、语气如何。

定中心：章节原句线 / 语言语气线。

第一路径：全文词位精确定位。

```text
完整原词/原句全文检索
-> chapters.full_text 真源定位
-> segments 上下文
-> 必要时 literary_texts_axis / evidence_edges 补语境
```

材料升级条件：必须能给出原句所在回目、说话人或叙述位置、前后文语境。

降级条件：脱离上下文解释词义，或只凭现代词义解释，不能作最终裁判。

## 策略模板 07｜贡献 / 作用 / 影响题

触发题型：某人贡献了什么、某物起什么作用、某事件对后文有什么影响。

定中心：因果机制线 / 功能作用线。

第一路径：事件链和证据边。

```text
主体归一
-> event_segments / evidence_edges 查动作与后果
-> person_segment_edges / objects_axis / spaces_axis 补参与对象
-> chapters.full_text 串前因后果
```

材料升级条件：必须有动作、结果、后续回响或他人评价形成链条。

降级条件：只出现某人某物但没有作用过程，不能回答贡献或影响。

## 策略模板 08｜忌讳 / 避讳 / 禁忌 / 规矩题

触发题型：问某人忌讳什么、避开什么、什么话不能说、什么事犯规矩。

定中心：言行边界 / 规训语境中心。

第一路径：原文词位 + 场景裁判。

```text
忌讳 / 避讳 / 不许 / 不可 / 怕 / 犯 / 规矩 / 体统 / 礼 等词位
-> 人物或场景轴锁定主体
-> segments 读说话现场
-> chapters.full_text 判断这是个人忌讳、家族规矩、礼法禁忌还是叙述反讽
```

材料升级条件：原文必须说明禁止、害怕、回避、责备或后果。

降级条件：现代读者推测、泛泛性格判断、单个情绪词，不能直接当忌讳。

## 策略模板 09｜空间 / 场景题

触发题型：在哪里、某场景里有什么人/物/事、某空间的功能。

定中心：空间场景中心。

第一路径：空间轴。

```text
spaces_axis / space_evidence_axis
-> person_segment_edges / objects_axis / event_segments
-> segment_range_edges 连续场景
-> chapters.full_text 场景裁判
```

材料升级条件：地点、人物、物象、事件在同一连续场景中形成可读关系。

降级条件：只命中地点名但没读清场景功能，只能作定位。

## 策略模板 10｜自述 / 亲口说 / 梦境题

触发题型：某人是否亲口说、自己承认、自己说梦、醒后复述、梦中说话。

定中心：说话边界 / 叙述层级中心。

第一路径：原文句和说话层级裁判。

```text
人物归一
-> 原文词位定位“说、道、梦、醒、告诉”等
-> 坐标或段落定位候选
-> chapters.full_text 区分叙述者说明、梦中台词、醒后自述、他人转述
```

材料升级条件：必须能证明“本人醒后/当场亲口说”这一层级。

降级条件：叙述者说某人做梦、梦中台词、他人转述，不等于本人醒后自述。

## 常用问题模板族

### G族｜贡献 / 作用 / 影响题

主模板：模板 07。

G族解决的是“谁对什么结果起了什么作用”。

#### G1｜A 单主体贡献

触发问法：A贡献了什么，A起了什么作用，A对某事有什么影响。

两两可查：A + 结果 / A + 事件 / A + 评价。

执行顺序：

```text
A 归一
-> A 的行动 / 发言 / 安排 / 选择
-> event_segments / evidence_edges 查动作与后果
-> chapters.full_text 串前因后果
```

升级条件：必须能看到动作、结果、后续回响或他人评价。

#### G2｜AB 共同贡献

触发问法：A和B共同贡献了什么，A/B分别起什么作用，二人如何共同造成某结果。

两两可查：A + 结果；B + 结果；A + B 同场 / 协作 / 对照。

子问题策略队列示例：

```text
1. 子问题：A 对结果 R 有什么作用｜两两可查：A + R｜重新选择策略：模板07
2. 子问题：B 对结果 R 有什么作用｜两两可查：B + R｜重新选择策略：模板07
3. 子问题：A 与 B 是否在同一行动链里协作或互相影响｜两两可查：A + B｜重新选择策略：模板02
```

降级条件：只证明 A 和 B 都出现，不等于共同贡献。

#### G3｜ABC / 多人贡献

触发问法：三人或多人分别贡献什么，谁贡献最大，几个人共同完成了什么。

必须进入模板00拆子问题。

队列形状：

```text
1. A + 结果 R
2. B + 结果 R
3. C + 结果 R
4. A/B/C 共同场景或连续事件链
```

升级条件：每个人的贡献必须单独过账；多人合成结论只能在各子问题都有证据后形成。

#### G4｜多情景贡献

触发问法：A在几个场景里分别起什么作用，A在前后不同情景中的作用如何变化。

两两可查：A + 场景1；A + 场景2；A + 场景3。

执行顺序：先按场景分组，再逐场景回原文；不得把一个场景的作用推广到所有场景。

#### G5｜多条件贡献

触发问法：在某条件下 A 是否有贡献，A在条件 X/Y/Z 下的作用是否不同。

两两可查：A + 条件X；A + 条件Y；A + 结果R。

升级条件：原文必须能证明条件成立，再证明贡献成立；条件不成立时只能降级为旁证或反证。

### X族｜交集 / 共线 / 两两同场题

主模板：模板 02、03、04、09；复杂时用模板00。

X族解决的是“两个对象是否在同一条原文线、场景线、事件线或文本线里共同成立”。

#### X1｜人物 + 人物共线

两两可查：人物A + 人物B。

优先库线：人物归一 -> person_segment_edges -> evidence_edges -> segments / chapters.full_text。

升级条件：同段、同场、对话、动作、评价、影响之一成立；同回不同场不升格。

#### X2｜人物 + 物象共线

两两可查：人物A + 物象B。

优先库线：人物归一 -> objects_axis -> evidence_edges / person_segment_edges -> 原文。

升级条件：出现、同场、接触、使用、经手、赠与、观看等阈值必须分清。

#### X3｜人物 + 空间 / 场景共线

两两可查：人物A + 空间B / 场景B。

优先库线：spaces_axis / space_evidence_axis -> person_segment_edges -> segment_range_edges -> 原文。

升级条件：人物确实进入、在场、行动或被该空间功能影响；只提地点名不算。

#### X4｜人物 + 时间 / 季节共线

两两可查：人物A + 时间B / 季节B。

优先库线：time_blocks / chapter order -> person_segment_edges -> segments -> chapters.full_text。

升级条件：时间字段、节令词和人物行动必须在同一原文现场或连续场景中成立。

#### X5｜物象 + 空间 / 事件共线

两两可查：物象A + 空间B；物象A + 事件C。

优先库线：objects_axis -> spaces_axis / event_segments -> evidence_edges -> 原文。

升级条件：物象不是泛意象，必须在空间或事件中有可读功能。

#### X6｜多轴交集

触发问法：A、B、C同时交集；人物、物、空间、时间一起成立。

必须进入模板00拆子问题。

队列形状：

```text
1. A + B
2. A + C
3. B + C
4. A + B + C 的同原文现场裁判
```

降级条件：两两都成立，不自动等于三元同场；最后必须回原文确认三者同现场。

#### X7｜首次交集

主模板：模板 03 + 模板 04。

两两可查：人物A + 对象B；再按章回 / 段号 / 字位排序。

升级条件：必须写明成立阈值，是出现、看见、接触、使用、经手，还是象征意象。

### P族｜诗词文共线题

主模板：模板 05；和人物、物象、空间、命运交叉时叠加模板 02/03/09/07。

P族解决的是“诗词文不是孤立文本，它和人物、物象、场景、事件、命运怎么共线”。

#### P1｜章回 + 诗词文清单

触发问法：某一回有没有诗词曲歌赋判词对联。

两两可查：chapter_no + literary_texts_axis。

执行顺序：chapter_no -> literary_texts_axis -> text_type/title/theme_tags -> chapters.full_text 抽检。

这是今天第五回诗词文题的优先模板。

#### P2｜人物 + 诗词文

触发问法：某人物有哪些诗词、判词、曲文、题咏；某诗词和某人物有什么关系。

两两可查：人物A + 文本T。

优先库线：人物归一 -> literary_texts_axis 的 author / target_person / related_person / theme_tags -> evidence_edges -> 原文。

升级条件：必须区分作者、被判对象、被提及对象、象征指向、旁观者；不能把诗中意象直接当人物事实。

#### P3｜物象 + 诗词文

触发问法：某物在诗词里如何出现，某诗词和某器物/植物/意象有什么关系。

两两可查：物象A + 文本T。

优先库线：objects_axis -> literary_texts_axis 原文 / theme_tags / imagery -> evidence_edges -> chapters.full_text。

升级条件：必须区分实物、诗词意象、象征物、标题字；诗词意象不能自动等同于现实接触。

#### P4｜空间 / 场景 + 诗词文

触发问法：某场景里有哪些诗词，某诗词发生在哪个空间，诗社/宴席/梦境中的诗词文本。

两两可查：空间A + 文本T；场景A + 文体T。

优先库线：spaces_axis / scene_groups -> literary_texts_axis -> segments -> chapters.full_text。

升级条件：诗词文本必须和该空间或连续场景同源，不可只因同回出现就并入。

#### P5｜诗词文 + 命运 / 伏笔 / 贡献

触发问法：某判词或曲文预示谁，某诗词对理解人物命运有什么作用。

两两可查：文本T + 人物A；文本T + 后文事件R。

优先库线：literary_texts_axis -> prophecy/evidence_edges -> event_segments -> chapters.full_text。

升级条件：必须有文本原句、对应人物/事件、后文回响；不能只凭现代阐释。

#### P6｜人物 + 物象 + 诗词文

触发问法：某人物和某物在诗词里是否共线，某人物借某物写诗/被判词绑定。

必须按模板00拆队列：

```text
1. 人物A + 文本T
2. 物象B + 文本T
3. 人物A + 物象B 是否在文本或原文场景中共同成立
```

### C族｜场景 / 条件 / 过滤题

主模板：模板 09；复杂时用模板00。

#### C1｜单场景盘点

触发问法：某场景里有什么人、物、事、诗词。

队列：场景 + 人物；场景 + 物象；场景 + 事件；场景 + 诗词文。

#### C2｜多场景比较

触发问法：几个场景分别发生了什么，有何不同。

必须拆成每个场景一条队列，再做对照；不得先写总体印象。

#### C3｜条件过滤

触发问法：只查某回、某季节、某地点、某人物在场、某物出现时的材料。

队列：主体 + 条件；条件 + 原文锚点；主体 + 结果。

升级条件：条件先成立，材料再成立。

### F族｜首次 / 顺序 / 阈值题

主模板：模板 04。

常见阈值：第一次出现、第一次同场、第一次接触、第一次使用、第一次亲口说、第一次被他人评价。

队列：主体 + 对象；主体 + 阈值动作；按 chapter_no / segment_no / start_char 排序；逐条回原文裁判。

硬规则：必须先声明阈值，否则不能回答“第一次”。

### N族｜否定 / 反证 / 有没有题

主模板：按对象选择 01-10，必要时叠加模板04或00。

触发问法：有没有、是否从未、是不是没有、能不能证明。

队列：正向召回 + 反向排除 + 降级材料说明。

升级条件：能回答“有”的必须有原文主证；回答“没有/未见”必须说明覆盖范围和检索边界。

### Q族｜原句 / 词义 / 版本真源题

主模板：模板 06。

常见问法：某词什么意思、这句话是谁说的、某词是否原文有、繁简异文、上下文语气。

队列：完整词组 + 章节真源；拆词组合 + 同句上下文；异体/繁简 + 版本真源。

硬规则：完整词组优先于拆词；拆词只能补漏，不能替代原句。

### R族｜关系 / 情感 / 亲疏 / 称谓题

主模板：模板 02；涉及称谓归一时叠加模板 01；涉及发展变化时叠加模板 04。

R族解决的是“关系不是标签，而是动作、称谓、场景和前后变化共同构成”。

常见问法：A和B是什么关系，二人关系如何变化，谁对谁亲近/疏远，某个称谓说明什么。

队列：A + B；A + 称谓/身份；A + B + 场景；前状态 + 后状态。

升级条件：必须有同场、对话、动作、评价、称谓或事件后果之一；关系变化题必须按章回顺序排。

降级条件：只同回、只被旁人提及、只凭现代情感词，不升关系结论。

### E族｜事件 / 情节 / 因果链题

主模板：模板 07；需要顺序时叠加模板 04；多事件时进入模板00。

E族解决的是“一件事如何发生、转折、造成结果”。

常见问法：某事为什么发生，某事造成什么后果，前文事件如何影响后文。

队列：原因C + 事件E；事件E + 结果R；事件E + 参与人物A；前事件E1 + 后事件E2。

升级条件：必须看见起因、过程、转折、结果或后续回响；时间先后不能自动当因果。

降级条件：只抓到事件标题、摘要、相邻章回，不足以回答因果。

### B族｜比较 / 对照 / 强弱 / 排行题

主模板：按对象选择 01-10；多对象比较默认考虑模板00。

B族解决的是“比较必须先定共同维度”。

常见问法：A和B有什么不同，谁更重要，哪个证据更强，几个人贡献大小如何。

队列：A + 维度D；B + 维度D；A + B 同场/同维度对照；评判标准 + 候选集合。

升级条件：A/B必须在同一维度上有可并列材料；排行题必须写清评价尺度。

降级条件：一边有强证一边只有旁证，不能硬排高低。

### S族｜象征 / 意象 / 主题功能题

主模板：模板 03、05、07、09；抽象主题复杂时进入模板00。

S族解决的是“象征解释必须落回功能和原文动作”。

常见问法：某物象象征什么，某空间有什么意味，某诗词意象如何指向人物命运。

队列：物象O + 人物/事件A；空间S + 事件E；诗词文本T + 意象I；主题词K + 原文承载点。

升级条件：必须有出现段落、人物反应、情节后果、文本原句或主题复核；不能只空谈象征。

降级条件：只因同字、同色、同类意象相似，不升主题结论。

### M族｜命运 / 结局 / 判词 / 伏笔 / 呼应题

主模板：模板 05、07；涉及前后顺序叠加模板 04；复杂时进入模板00。

M族解决的是“判词曲文、前文伏笔、后文结局之间如何互证”。

常见问法：某判词预示谁，某人物结局如何被前文伏笔照应，前后回如何呼应。

队列：人物A + 判词/曲文T；人物A + 结局事件R；前文线索F + 后文兑现R；文本T + 命运节点M。

升级条件：必须同时有文本原句、人物归属、后文事件或收束段落；现代解释不能替代原文。

降级条件：只凭判词一句不回后文，或只凭结局不回前文伏笔，不能作完整命运链。

### V族｜视角 / 叙述 / 作者写法题

主模板：模板 06、10；涉及结构呼应时叠加 M族或 S族。

V族解决的是“谁在说、谁在看、叙述者怎样组织材料”。

常见问法：这句话是谁说的，作者为什么这样写，这是不是反讽/留白/伏笔。

队列：原句Q + 说话人/叙述者；视角主体A + 场景S；叙述提示N + 前后结构；反讽命题J + 反证对象。

升级条件：必须定位原句、说话层级、前后文结构；作者意图题要落回文本组织。

降级条件：读者猜想、现代评论、生成稿解释，不能替代原文结构。

### L族｜清单 / 全查 / 覆盖 / 穷尽题

主模板：按对象选择 01-10；全书范围叠加全文穷尽；多轴清单进入模板00。

L族解决的是“哪些、几次、所有、全书”类问题的覆盖边界。

常见问法：全书有哪些，出现几次，哪些回，是否从未出现，所有重要场景是什么。

队列：对象O + 全文词形；对象O + 结构库轴；命中候选 + 原文上下文；正向召回 + 反证排除。

升级条件：必须写覆盖范围、词形范围、库轴范围和漏查风险；回答“没有/未见”必须说明检索边界。

降级条件：只查一个库、只查一种字形、只查现代词，不能说全书穷尽。

### D族｜差异 / 变化 / 阶段题

主模板：模板 04；涉及关系变化叠加 R族，涉及事件变化叠加 E族。

D族解决的是“前后状态、阶段、变化原因”。

常见问法：A前后有什么变化，某关系经历几个阶段，某空间功能怎样变冷/变空。

队列：主体A + 阶段1；主体A + 阶段2；阶段1 + 阶段2 对照；变化点 + 原因/后果。

升级条件：必须按章回顺序，写出变化阈值和变化触发点。

降级条件：只拿单点材料回答变化题，或把时间先后误当状态变化。

### H族｜群像 / 类别 / 人物组题

主模板：模板 01、02、05、07；复杂时进入模板00。

H族解决的是“一个群体如何有共同点，又如何有差异”。

常见问法：十二钗群像如何展开，几个丫鬟功能有何不同，同类物象在全书如何分组。

队列：成员A + 共同维度D；成员B + 共同维度D；成员C + 共同维度D；群体 + 差异维度。

升级条件：每个成员必须单独过账；群体结论必须说明共同点和差异点。

降级条件：把一个人物的材料套到整个群体，或只列清单不做关系/功能判断。

### O族｜物件流转 / 赠与 / 经手 / 分配题

主模板：模板 03、07；涉及多人多物时进入模板00。

O族解决的是“物从哪里来、经谁手、到哪里去、发挥什么功能”。

常见问法：谁给谁什么，某物如何流转，某物在哪个事件里改变关系。

队列：物象O + 持有人A；物象O + 接收人B；A + B 赠与/拒收/经手；物象O + 后果R。

升级条件：必须有拿出、赠与、接收、拒收、使用、保存、毁弃等动作；同场不等于流转。

降级条件：只因物和人在同回出现，不能说经手或赠与。

### Z族｜制度 / 礼法 / 家族秩序题

主模板：模板 07、08；涉及人物关系叠加 R族。

Z族解决的是“规矩、礼法、权力如何通过人物行动执行”。

常见问法：某场景体现什么家规，某婚姻/礼法压力如何作用到人物，谁执行了制度。

队列：制度词K + 场景S；人物A + 执行动作；人物B + 被影响结果；规矩/礼法 + 原文后果。

升级条件：必须有制度词、身份关系、执行动作或后果；抽象制度概念不能单独成证。

降级条件：只讲礼法概念，不看具体场景执行。

### 30题抽象出的执行心得

```text
1. 复杂题先不求答案，先求问题形状。
2. 一个问题超过两类对象，优先判断是否进入模板00。
3. 子问题必须是两两可查，不是文章小标题。
4. 每个子问题都重新选择策略模板。
5. 比较题先定共同维度。
6. 象征题先找现实动作、人物反应或文本原句。
7. 命运题必须同时找判词/伏笔和后文兑现。
8. 关系题必须有同场、动作、称谓、评价或后果。
9. 全书题必须写覆盖边界和漏查风险。
10. 聚拢头和坐标头共用策略选择；差别只在后端如何取材和裁判。
11. 策略卡必须拿出来，但卡只是起手方法；证据反馈后允许组合、换卡或有理由偏离。
12. 偏离模板不是违规；没有策略、没有理由、没有回原文裁判才违规。
```

### 模板族选择提示

```text
1. 能单模板解决，就选 01-10，并标注模板族。
2. 一题里出现多人、多物、多场景、多条件、多阈值，先考虑模板00。
3. 诗词文只要和人物、物象、空间、命运相连，就按 P族生成两两队列。
4. 贡献题只要超过一个主体、一个情景或一个条件，就按 G族拆队列。
5. 交集题必须写明两两对象；三元以上交集必须有最终同现场裁判。
6. 关系、称谓、情感、亲疏题先按 R族拆动作和场景，不能只拿标签当关系。
7. 因果、贡献、影响题先按 E族/G族找过程和结果，不能把时间先后当因果。
8. 比较、排行、主要/次要题先按 B族定共同维度，不能一边强证一边旁证硬比。
9. 象征、主题、命运题先按 S族/M族落回原文动作、文本原句和后文兑现。
10. 全书、哪些、几次、有没有/未见题先按 L族/N族写覆盖边界。
```

## 模板选择顺序

AI 进入查询逻辑阶段、定执行顺序时，按下列优先级选模板：

```text
1. 明确问原句/词义：先用模板 06。
2. 明确问诗词文：先用模板 05。
3. 明确问人物身份/别名：先用模板 01。
4. 明确问人物关系/同场/交流：先用模板 02。
5. 明确问人物与物象交集：先用模板 03；若问首次，再叠加模板 04。
6. 明确问贡献/作用/影响：先用模板 07。
7. 明确问忌讳/规矩/避讳：先用模板 08。
8. 明确问空间场景：先用模板 09。
9. 明确问亲口说/自述/梦：先用模板 10。
10. 多轴混合题：如果能确定主问题，先定主模板，再把其他模板作为补查或降级条件。
11. 如果无法确定主模板，必须选择模板 00｜拆子问题策略，并写出 Codex子问题清单和 Codex子问题策略队列。
```
