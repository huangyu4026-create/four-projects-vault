# 000D_经验复盘写回_新窗口强制入口

生成时间：2026-06-22

性质：经验复盘触发头 / 规则写回入口 / 学习心得入账门

状态：候选正式头；不改旧库；不自动写 SQLite；用于把用户确认过的查询经验写回工程学习层

## 一句话定案

用户说到下列触发词时：

```text
经验
经验值
经验复盘
经验回盘
经验提取
经验总结
经验入账
增加经验值
写入经验
把这个经验记下来
```

AI 必须进入本文件规定的经验复盘写回流程。

本入口解决的问题不是重新回答题目，而是把已经验证、已经被用户确认有价值的查询经验，写回到以后能被 AI 调用的规则层。

## 总铁律

经验写回必须先判定经验身份。

```text
已验证经验：可以写入对应规则文件或学习索引。
候选经验：只能写入候选池或本题复盘包，不能升格成硬规则。
个人反馈：只写本题复盘，不改总规则。
未验证猜想：不写规则，只列为待验证。
```

经验不是最终答案，不替代材料池四态、精读材料词和写作前原文追证。

经验写回不许直接修改 SQLite、原始 CRV 迁移底座或在线 Notion。

## 触发后第一动作

听到触发词后，第一步不是写文件，而是回答短卡：

```text
已触发经验复盘。
经验对象：____
经验状态：已验证 / 候选 / 个人反馈 / 未验证猜想
拟写入位置：____
是否需要用户确认：是 / 否
```

如果用户已经明确说“这个经验确定”“写进去”“记下来”“总结得很好，把它写回”，可以直接进入写回；否则先报告拟写入位置和摘要。

## 写回分流规则

经验必须按用途写到正确位置。

| 经验类型 | 写回位置 | 用途 |
|---|---|---|
| 查询逻辑策略模板 | `000E_查询逻辑策略模板组_新窗口学习入口.md` | 位于取词之后，影响定中心、定路径、定词网后的查证顺序 |
| 坐标查询拆题 / 取词 / 库线优先级 | `000C_进入坐标查询_新窗口强制入口.md` | 影响以后进入坐标查询时如何拆题 |
| 聚拢查询 / 旧闭环路线 / 入口秩序 | `000_进入红楼梦工程_新窗口强制入口.md` 或 117/119/120 等聚拢规则文件 | 影响旧头路线 |
| 通用经验复盘流程 | `000D_经验复盘写回_新窗口强制入口.md` | 影响以后如何写经验 |
| 单题成败原因 | 本题包 `00E_经验复盘入账.md` / `00F_经验复盘入账.json` | 留存本题复盘 |
| 可复用但未升格规则 | 查询法典候选池 / 学习心得索引 | 先候选，待重复验证 |
| 结构化学习心得 | `outputs/正式底库本地专门索引/04_学习心得索引_20260619.*` | 便于后续检索与回挂 |

## 写回前五问

每条经验写回前，必须完成五问：

```text
1. 这条经验来自哪次具体查询？
2. 成功或失败的客观证据是什么？
3. 它影响的是题型识别、查询策略、库线顺序、证据裁判，还是最终表达？
4. 它应该成为硬规则、候选规则，还是只留本题复盘？
5. 下次触发它的题型或关键词是什么？
```

五问不清楚时，只写候选，不写硬规则。

## 写回格式

写入规则文件时，使用短小动作规则，不写长篇感想。

推荐格式：

```text
## 经验规则：____

触发题型：
____

规则短句：
____

执行顺序：
1. ____
2. ____
3. ____

降级条件：
____

回原文裁判：
____
```

写入本题复盘包时，保留：

```text
问题
原路线
修正路线
成功信号
失败信号
关键词经验
库线经验
下次动作
```

## 坐标查询经验写回特别规则

坐标查询的经验，优先写回 `000C`，因为 `000C` 控制 AI 读题取词和坐标取材第一动作。

例如诗词文题：

```text
触发题型：问某回、某人、某场景有没有诗、词、曲、歌、赋、判词、对联、题咏等内容。
规则短句：先用 chapter_no / 人物 / 场景锁定 literary_texts_axis / 诗词文库，再用全文词位和变量投影补漏，最后回 chapters.full_text 抽检关键段。
降级条件：诗词文库只给结构化索引，不能替代原文裁判；遇到异文、重复分类、边界不明时必须回原文。
```

这类经验不应只写入一次性坐标材料包，因为一次性包不会稳定影响下一次 AI 拆题。

## 状态词

```text
经验已触发 = 已识别经验复盘请求。
经验待判 = 已收到经验，但尚未判断可否写回。
经验候选已留存 = 已写入候选池或本题复盘，不是硬规则。
经验已写回 = 已写入对应规则文件或学习索引。
经验硬阻断 = 缺少验证证据、缺少用户确认，或写回位置不明。
```

## 禁止事项

```text
不因一次成功就随意改硬规则。
不把用户随口评价直接升格为法典。
不把候选材料当作已验证经验。
不写入 SQLite。
不覆盖旧经验，只追加带日期的新规则。
不把经验写进最终答案正文冒充证据。
```

## 给新线程的一句话

如果用户说：

```text
经验复盘：……
```

新线程必须理解成：

```text
进入经验复盘写回。
先判断经验是否已验证。
再判断写回位置。
能升格的写规则文件。
不能升格的写候选池或本题复盘。
```
