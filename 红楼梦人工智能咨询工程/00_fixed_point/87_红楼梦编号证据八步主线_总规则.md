# 87_红楼梦编号证据与现场裁决主线_总规则

生成时间：2026-06-20
状态：由“编号证据八步”升级为“编号证据底层法 + 现场裁决九步正式主线”

## 一句话结论

旧八步负责找到证据位置；新九步负责判断现场真相。

正式主线是：

```text
归一 -> 收点 -> 交集找候选 -> 成场 / 路由门打开现场 -> 分类关系类型 -> 判在场等级 -> 判场点性质 -> 回原文强场 -> 入材料池 / 输出
```

## 为什么必须升级

旧工程容易滑向：

```text
猜搜索词 -> 搜文本 -> 堆材料 -> 写答案
```

编号八步把它改成：

```text
确认对象 -> 收编号 -> 做交集 -> 打开原文窗口 -> 分清角色 -> 回原文裁判 -> 凭证入池 -> 写答案
```

但现在场库出现以后，只做到“打开窗口”还不够。

因为红楼梦问题经常问的是：

```text
谁和谁是否同场？
一个物件在什么现场发挥作用？
一个人物是不是亲自到场？
某个事件是发生、转述、回忆、命令还是象征？
一组编号背后到底是不是同一个现场？
```

所以必须增加现场层：

```text
编号证据 -> 现场候选 -> 在场等级 -> 场点性质 -> 原文强场
```

## 旧八步与新九步的关系

| 层级 | 作用 | 是否保留 |
|---|---|---|
| 编号八步 | 找对象、收编号、做交集、开窗口 | 保留为底层能力 |
| 现场九步 | 判关系、判在场、判场点、强场入池 | 升为正式主线 |
| 场面库 | 提供回目、时间块、场组、场点的现场骨架 | 接入主线 |
| 原文 | 最终裁判 | 永远最高 |

## 现场裁决九步

| 步骤 | 名称 | 人话解释 | 必须产物 |
|---|---|---|---|
| S1 | 归一 | 确认用户说的是谁、什么物、什么空间、什么事件、什么词义。 | 对象清单、规范名、别名、查询词、character_key。 |
| S2 | 收点 | 把对象在全书里的位置收成编号。 | segment_no、line_id 或 line_id_pending、cluster_unit、event_id、chapter_no、scene_id 候选。 |
| S3 | 交集找候选 | 看对象之间有没有同点、同窗口、同事件、同场组、同回目。 | 交集表、强弱等级、候选现场。 |
| S4 | 成场 / 路由门 | 从编号打开周边现场，必要时临时合成聚拢场。 | 现场窗口、人物、空间、物象、活动、前后场。 |
| S5 | 分类关系类型 | 判断对象之间是什么关系。 | 同场、说话、送物、路线、转述、象征、反证、误召回等分类。 |
| S6 | 判在场等级 | 判断这个关系离真实现场有多近。 | real_presence / mentioned_only / memory_or_recall / remote_influence / negative_evidence / uncertain_presence。 |
| S7 | 判场点性质 | 判断这个现场的工程重量。 | focus_scene / functional_scene / passing_scene / minimal_mark / side_route_only。 |
| S8 | 回原文强场 | 带着问题和现场候选回原文裁判。 | 原文锚点、原文摘录、上下文判断、强弱证据。 |
| S9 | 入材料池 / 输出 | 把完成裁决的材料入池，或直接形成简单答案。 | 材料池条目、subquestion_refs、证据等级、补证状态、可写结论。 |

## 关键区别：交集、路由、分类、判在场不是一回事

### 交集

交集只负责找门。

它回答：

```text
哪些编号、窗口、事件、场组值得打开？
```

### 路由门 / 成场

路由门负责开门。

它回答：

```text
这个编号周边发生了什么？
谁在？哪里？什么动作？前后是什么场？
```

### 分类关系类型

分类负责看屋里发生了什么。

它回答：

```text
这些对象之间是什么关系？
是同场、说话、送物、路线、被提及、回忆、象征还是反证？
```

### 判在场等级

判在场负责给现场定真假强弱。

它回答：

```text
这能不能算真实在场？
是不是只是被提及？
是不是回忆、梦境、远端命令、物件影响？
```

### 判场点性质

判场点性质负责给材料定重量。

它回答：

```text
这是焦点场、功能场、过场、最低标注，还是旁路材料？
```

## 八步之前：问题树层

用户问题进来以后，不是直接进入九步。

先判断它是：

```text
简单题
复杂题
抽象题
全文穷尽题
```

八步法本身是完整方法。在八步法基础上，先完成取词之前的思考：

```text
读题 -> 判断题型 -> 拆对象 -> 定主轴 -> 定查询词策略 -> 取主查词 / 归一词 / 强复合轴 / 扩展词 / 背景词 / 排除词
```

取词形成入口词包以后，进入“取词以后如何用库”的思考：

```text
带着入口词包 -> 判断库轴 -> 收点 -> 交集 -> 成场 / 路由门 -> 分类 -> 判在场 -> 判场点 -> 回原文 -> 入材料池
```

硬规则：

```text
八步法覆盖取词前思考和取词后用库思考。
取词前要想清楚题型、对象、主轴、查询词策略和入口词。
取词后要想清楚用哪个库轴、怎样收点、怎样交集、怎样回原文、怎样入材料池。
不能跳过八步法，直接搜库、直接写答案。
```

简单题不强拆，默认一个问题单元。

复杂题、抽象题、长文题必须拆成问题单元。

每个问题单元都要能接上九步主线：

```text
子问题 -> 归一 -> 收点 -> 交集找候选 -> 成场 / 路由门 -> 分类关系 -> 判在场 -> 判场点 -> 回原文 -> 入材料池
```

但不是每个子问题都机械重跑全文。

允许共享：

```text
同一人物归一包
同一物象收点
同一 segment_no
同一 cluster_unit
同一 scene_id
同一 scene_group_id
同一 event_id
同一原文窗口
```

共享材料必须写明：

```text
subquestion_refs
```

## 子问题出卷闸门

复杂题被拆成几个子问题，就必须逐个子问题过账。

每个子问题至少要有：

```text
子问题编号
子问题原文
对应材料池条目
材料可用性
原文锚点
现场候选
在场等级
场点性质
当前覆盖状态
对子问题的临时回答
是否需要补证
```

覆盖状态只能是：

```text
已覆盖
部分覆盖
暂缺
不成立 / 无强证
```

硬规则：

```text
子问题不过账，不准出卷。
总答案不得替代子问题回答。
材料池必须按子问题分别归类。
一个材料可以服务多个子问题，但必须写明 subquestion_refs。
```

## 补证循环

补证循环不是重新整题乱跑。

它必须生成补证任务卡：

```text
为什么补证？
补哪个子问题？
缺的是人物、物象、空间、事件、现场、原文上下文、反证还是解释链？
应该回哪个库、哪个编号、哪个场点、哪个回目、哪个原文窗口？
本轮补证后是否允许写作？
```

补证只让受影响的问题单元回到：

```text
归一 -> 收点 -> 交集找候选 -> 成场 / 路由门 -> 分类关系 -> 判在场 -> 判场点 -> 回原文
```

## 补证触发条件

允许触发：

```text
材料池判定 writing_mode = 先补证。
写作前原文追证 writing_permission = 先补证。
某个关键子问题 coverage_status = 暂缺或部分覆盖。
强结论只有摘要定位，没有原文足证。
现场候选无法判断 real_presence。
同回共现可能被误写成同场。
提及、回忆、远端影响可能被误写成真实在场。
终稿复核发现强证不足。
```

不允许触发：

```text
只是为了显得更复杂。
没有明确缺口。
补证方向只是空泛地说“再查一下”。
已经没有新编号、新场点、新回目或新原文窗口。
简单事实题已经真源一次命中。
```

## 补证轮次原则

常规题：

```text
最多一轮补证。
```

复杂题：

```text
可以二到三轮补证。
```

超复杂长文题：

```text
可以继续补证，但每一轮必须有新对象、新编号、新场点、新原文窗口或新反证。
```

如果连续一轮没有新增有效证据，就停止循环：

```text
写谨慎答案
或输出补证说明
或标明当前不能下强结论
```

## 每一步失败红线

### S1 归一

```text
不能只按用户原词查。
不能有人物线索但不做人名消歧。
不能把称谓、别名、规范名当成不同对象。
```

### S2 收点

```text
不能只有搜索词命中，没有编号。
不能没有来源说明。
不能把候选文本直接当证据。
```

### S3 交集找候选

```text
不能把同回共现直接写成同场。
不能不区分 segment_no、cluster_unit、event_id、scene_id、chapter_no。
不能没有交集却硬写事实关系。
```

### S4 成场 / 路由门

```text
不能只看一个命中句就断定现场。
不能不看前后窗口。
不能忽略空间、人物、物象、活动和前后场。
```

### S5 分类关系类型

```text
不能把提及、回忆、象征、命令、物件影响都混成同场。
不能不说明关系性质。
```

### S6 判在场等级

```text
不能把 mentioned_only 写成 real_presence。
不能把 memory_or_recall 写成现场发生。
不能把 remote_influence 写成亲自到场。
```

### S7 判场点性质

```text
不能把过场夸大成焦点场。
不能把旁路材料放成主证。
不能不说明材料重量。
```

### S8 回原文强场

```text
不能只读摘要不回原文。
不能脱离上下文摘一句。
不能用AI常识替代原文裁判。
```

### S9 入材料池 / 输出

```text
不能无凭证入池。
不能材料池不分子问题。
不能强弱证据混放。
不能跳过补证状态。
```

## 一句话收束

编号主线让 AI 找得到门。
现场裁决让 AI 知道门里面到底发生了什么。
原文强场让答案站得住。
