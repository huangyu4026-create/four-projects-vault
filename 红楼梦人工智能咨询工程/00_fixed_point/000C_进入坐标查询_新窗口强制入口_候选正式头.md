## 现场交互修正活账硬锁

坐标工程允许人机中途交流，但交流不是另开聊天线，而是一次状态回流。触发词不是死按钮，也不是封闭词表；它只是帮助 AI 看清当前讨论焦点。只要用户中途围绕“子问题 / 拆分 / 取词 / 词网 / 策略 / 方法 / 模板 / 执行顺序 / 补证 / 材料不够 / 四态 / 放行 / 重跑”等内容讨论，随后说“好的 / 同意 / 继续 / 继续修正 / 继续完善 / 进入工程再执行 / 再跑 / 重跑”等放行话，AI 不得直接继续往下跑。

必须先写或续写 `Codex现场修正活账`，并按前面真实讨论的焦点判定本轮回到哪个策略讨论节点：

```text
子问题策略讨论节点：子问题、拆分、拆题、两两对象、两两交集、问题树、问题拆成几问。
取词-查表策略讨论节点：取词、词网、强复合、词角色、策略、方法、模板、查证顺序、库线优先级、用表收点。
补证诊断门：材料不够、证据不够、补证、补查、内容不足、需要重跑材料。
四态放行诊断门：材料池、四态、放行、入池、精读材料词、写作前追证、最终答案。
```

其中补证诊断门和四态放行诊断门不是新根门；它们发现问题后，必须回流到子问题策略讨论节点或取词-查表策略讨论节点。若只能判断“材料不够”但说不清是拆分漏了还是取词查表策略错了，状态只能写“阻断待判”，不能继续写最终答案。

## AI取词硬规则

坐标工程不靠机器硬切自然句。

进入坐标查询以后，必须先由 AI 读题取词，再由坐标取材包流程验词、归一、查表、入材料池。

```text
自然问题
  -> AI 取词
  -> Codex查询词 / Codex强复合 / Codex词角色 / Codex材料升级条件
  -> Codex查询逻辑策略
  -> Codex查证顺序
  -> formal_honglou_coordinate_material_gate.material_pack_bundle()
  -> 材料池入口候选
```

硬规则：

```text
人物轴：人名不拆、不猜；必须先接 70/72 全人物归一包；变量投影里人物只按 variable_value_name 精确命中。
变量轴：先查 variable_value_name 精确命中；没有精确命中时，二字以上词才允许包含补救。
词位轴：明确正文词 / 短语 / 单字，先走全文词位库穷尽。
单字轴：单字分三层处理，不能一刀切。
  1. 物象根字可以保留：如竹可以引出竹子、竹叶、竹梢；金可以引出黄金、金器；花、泪、黄等可作物象/颜色/意象根轴。
  2. 明确问字词时可以保留：如“黄字”“金这个字”“眼泪的泪”。
  3. 人名拆字和宽网附带字不得自动升主查：如“黛玉”的“玉”、由“泪”宽泛带出的“珠”，只能作补查提示，不能冒充主轴。
代词轴：我、你、他、她、自己、本人不得作为主查词；它们只能作为说话边界、叙述视角或裁判条件。
同场轴：same_atom / same_scene / same_event 只表示候选接近，不等于最终结论。
原文轴：每个候选必须带 atom_code、segment_no、chapter_no、source_sentence。
```

## 坐标查询取词心法

坐标门不是裸搜工具。坐标工程有自己的八步思考和 Codex 查询词路：

```text
AI 先定中心。
AI 再定路径。
AI 再定词网。
AI 再定执行顺序。
工具只按这些令牌查库，不替 AI 猜词。
```

坐标门的 AI 取词必须完成六件事：

```text
1. 定中心：本题到底是人物、物象、空间、季节、事件、原句、距离、同场，还是多轴交叉。
2. 定主查词：只放原问题必须成立的核心对象。
3. 定强复合：把必须共同成立的对象放进 compound_groups，防止单边材料冒充关系证据。
4. 定词角色：区分主查词、归一词、扩展词、背景锚点、排除词。
5. 定查询词策略：说明中心词为什么入网，扩展词、背景锚点、排除词为什么保留或降级。
6. 定升级条件：什么材料可以入材料池，什么只能算候选或补查入口。
```

定完这六件事后，必须进入坐标工程查询逻辑策略经验模板组选择执行策略。这里不是“查询词策略”，而是取词之后的查询逻辑策略经验层：

```text
000E_B_坐标工程查询逻辑策略经验模板组_新窗口学习入口.md
```

也就是：先 AI 取词，再定中心、定路径、定词网，最后在坐标工程查询逻辑策略经验模板组里定执行顺序。人物题、人物+人物题、人物+物象交集题、贡献题、忌讳题、诗词文题等，都应先落入对应查询逻辑策略经验模板，再生成 Codex查证顺序。

强制字段：

```text
Codex查询词策略：中心词=____；强复合依据=____；扩展词=____；背景锚点=____；排除词=____；取舍理由=____
Codex查询逻辑策略：模板编号｜模板名称｜为什么选它
Codex结构转化表：题目中心=____；选择结构=人物结构/描写结构/推进结构/证据结构；查询编码=____；交叉办法=____；选择理由=____；下一步查证动作=____
```

如果所有策略都不合适，必须写：

```text
Codex查询逻辑策略：模板00｜拆子问题策略
Codex子问题：
1. ____
2. ____
Codex子问题策略队列：
1. 子问题：____｜两两可查：对象A + 对象B｜重新选择策略：模板__｜Codex查询词策略：____｜Codex查询词：____｜Codex强复合：____｜Codex查证顺序：____｜Codex材料升级条件：____
2. 子问题：____｜两两可查：对象A + 对象B｜重新选择策略：模板__｜Codex查询词策略：____｜Codex查询词：____｜Codex强复合：____｜Codex查证顺序：____｜Codex材料升级条件：____
```

模板00的子问题不能只列标题。子问题产生后必须分别排队，每个子问题重新进入 `000E_B` 的坐标工程查询逻辑策略经验模板选择；四个子问题就是四次策略选择。每条子问题必须写成两两可查的对象或强复合，再生成自己的查证顺序。

词的身份不可混：

```text
主查词：本题真正要证明关系的核心对象，进入 Codex查询词 和 Codex强复合。
归一词：人物别名、简称、繁简、异写；交给 70/72 人物归一或正字归一，不当第一轮执行词。
扩展词：同类物象、动作、状态、场景词，只用于补漏，不能自动升主证。
背景锚点：地点、回目、标题、名场面、诗词入口，只帮助定位，不能单独冒充答案。
排除词：容易串题、同字误召回、旧题污染、标题诱导和无关热词，用于降级或补查。
查询词策略：把以上取舍写成可复盘理由；只写词不写策略，等于没有证明词网为什么这样形成。
```

坐标门必须执行下列对象判断：

```text
人物对象：人名不拆、不猜。人物组只放归一人物实体，例如林黛玉；黛玉、林姑娘、颦儿等放归一词，由人物包处理。
物象对象：可抓最小有效物象字/词。竹可以作为竹子、竹叶、竹梢、竹影等物象根；金、玉、泪、帕、灯等单字物象也可以作主查词。
复合物对象：必须守住本体。竹扇可提示扇和竹两线，但普通扇不能自动算竹，必须靠原文或变量证据升级。
关系题：第一组通常是人物/主体，第二组是物象/空间/事件/季节；动作、情绪、地点、回目多为辅助，不轻易进强复合。
补查题：首轮单边、标题化、摘要化、材料互不支撑时，回到 AI 取词层调整词角色和强复合，不让程序自动扩大。
```

## 诗词文题取材专门规则

凡用户问某回、某人、某场景有没有诗、词、曲、歌、赋、判词、对联、题咏等诗词文内容，不要先把整回全文轮扫当作第一取材动作。

第一取材入口应优先走：

```text
literary_texts_axis / 诗词文库
```

执行顺序：

```text
1. 若题目带回目，先用 chapter_no 锁回。
2. 若题目带人物，再叠加 target_people / 人物变量。
3. 若题目带场景，再叠加 segment_no / scene / space / event 相关锚点。
4. 若题目带文体，再用 text_type / title / theme_tags 筛。
5. 诗词文库取到结构化清单后，再用全文词位和变量投影补漏。
6. 最后只回 chapters.full_text 抽检关键段，不把整回通读当第一动作。
```

词位与变量交叉在这里不是第一刀，而是补漏和裁判辅助：

```text
诗词文库负责先给结构化清单。
全文词位负责防漏：诗、词、曲、歌、赋、判词、对联、题曰、其判曰、其词曰等。
坐标查询中心库投影表/视图负责定点：poem / prophecy / event / object_fullscan_item / object_fullscan_class / space / person_identity 等。
chapters.full_text 负责最终原文裁判。
```

降级条件：

```text
诗词文库命中不等于最终答案。
同一文本可能被重复分类，如曲 / 红楼梦曲、判词 / 七绝 / 诗谶。
遇到异文、重复分类、标题化记录、边界不明时，必须回原文抽检。
```

坐标门的执行库：

```text
坐标门：AI取词 -> 000E_B策略选择 -> 坐标查询中心库（variable_points / v_variable_point_context）+ 词位库/字位桥表/距离共场 -> 原文裁判 -> 04入池凭证 -> 材料池
```

所以，坐标门不是“更聪明地切词”，而是“更硬地执行 AI 已经想清楚的词路”。

recall_gate：coordinate
Codex查询词策略：...

## 对象词提醒

对象词不要被单一路径锁死。

```text
全文词位负责防漏。
变量坐标负责定点。
变量未登记不等于原文没有。
精确变量也不能替代全文词位和原文裁判。
```

例如：

```text
花字：走全文词位 term_norm='花'
花物象：走变量 object_fullscan_item=花，必要时交叉 object_fullscan_class 和 object_fullscan_cooccurrence_candidate。
花相关扩展：由 AI 放在扩展词、背景锚点或补查方案中
```

最终靠人物、空间、季节、事件、距离 / 共场交叉裁判。

## 证据层级提醒

坐标候选必须标清证据层级：

```text
evidence_layer=原文句
evidence_layer=全文词位
evidence_layer=工程变量点:person_identity/object_fullscan_*/space/season/event

```

只有 `原文句` 可以直接进入原文裁判。

`工程变量点`、`段落摘要`、`工程推定` 都只能定位和组证，不能冒充原文。

坐标包必须同时输出：

```text

```

`可入池` 只表示候选可以交材料池四态，不表示可直接回答。

## 自述梦裁判门

遇到“某人自己说梦 / 亲口说做梦 / 醒后讲梦 / 自述梦”这类题，不新增大库，只加裁判门。

必须分四类：

```text
叙述者说明：叙述者说某人做梦。
梦中台词：人在梦里说话。
醒后自述：醒来以后本人说自己做了梦。
他人转述：别人说某人做梦或梦见什么。
```

只有“醒后自述”可以回答“本人亲口说我做了一个梦”。

第82回这种材料，坐标门可以快速定位梦境现场，但最终裁判必须回原文区分：

```text
有梦。
有梦中台词。
有醒后惊醒和反应。
不等于醒后本人亲口复述“我做了一个梦”。
```
