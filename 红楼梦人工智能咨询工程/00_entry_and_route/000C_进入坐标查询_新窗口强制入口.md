## 首轮子问题短卡硬闸

入口短报之后，第一道可见动作必须是 `SQ短卡`。这不是长笔记本，也不是事后复盘，而是开始查询前的最小通行证。

```text
SQ短卡：
SQ-01｜要回答的子问题｜对象A｜对象B/变量｜拟查入口表/编码｜状态=待查
SQ-02｜要回答的子问题｜对象A｜对象B/变量｜拟查入口表/编码｜状态=待查
SQ-03｜要回答的子问题｜对象A｜对象B/变量｜拟查入口表/编码｜状态=待查
本题总查法：两两交集 / 同场 / 距离 / 全文词位 / 原文裁判中的哪几种。
```

硬判定：

```text
没有先输出 SQ短卡，不得开始查库。
只说“我按四块来查”“我会分几类核”不算 SQ短卡。
用户事后追问策略本时补写出来的内容，只能叫事后审计版，不算现场短账。
```

简单题可以只有 1 条 SQ；复合题必须按问题自然分支列出多条 SQ。短卡要短，宁可一行一题，不写长文；但必须让用户和机器都看见当前正在回答哪一个子问题。

总答前还必须有 `SQ完成卡`。它也是短卡，不写长文，只证明每个子问题已经查完、裁判完、能不能进入总答：

```text
SQ完成卡：
SQ-01｜已查入口表/编码｜证据锚点｜结果状态=已确认/可喂下一步/只作弱证/不可喂下一步｜满意度=满意/不满意｜能否进入总答=是/否
SQ-02｜已查入口表/编码｜证据锚点｜结果状态=____｜满意度=____｜能否进入总答=____
总答放行：全部SQ可进入 / 部分SQ弱证保留 / 阻断待补。
```

没有 `SQ完成卡`，不得输出总答案；只在最后答案里自然分段回答，不算 `SQ完成卡`。

## 大库查询器：先分解，再查表

大库查询器不是新库，也不是第二入口。它就在坐标查询唯一中心大库内部，是机器进入大库前必须看的导航总表。

固定动作：

```text
1. 先读 coordinate_ai_query_master_matrix 的 usage_stage=入口总览，形成全库能力地图。
2. 把用户问题拆成 SQ-01 / SQ-02 / SQ-03...，每个子问题都要能落到一个 query_domain。
3. 对每个子问题写出 query_domain / query_intent / axis_a / axis_b / axis_c。
4. 回 coordinate_ai_query_master_matrix 取 primary_table / fast_table / detail_table / evidence_table / summary_table。
5. 再查 coordinate_smart_query_entry_map，确认正式入口表、辅助表和维护表，正式查询只走正式入口。
6. 总数题先 fast_table；证据题先 detail_table 或 evidence_table；复杂交叉题先 variable_points 或统一证据入口，再回源。
7. 命中以后按 source_back_fields 回 atom_code / scene_id / global_atom_order / 原文窗口裁判。
```

每个子问题进入查库前，必须形成一行大库查询器短卡：

```text
QMAP-00｜SQ=____｜主中枢=____｜条件中枢路线=____｜domain=____｜intent=____｜axes=____｜主入口=____｜快速表=____｜证据表=____｜回源字段=____
```

这张短卡的作用，是让机器先知道“用大库里的哪张表解决哪个子问题”。没有 QMAP 短卡，不能直接散搜。

空间/路线题的 QMAP 必须写清：`domain=空间/路线桥接` 时，主入口、快速表、证据表统一指向 `space_route_bridge_cards`；默认只取 `link_strength='strong'`，`review/unmatched` 只用于补漏和扩库。回源字段看 `source_table + source_id + atom_code + scene_id + global_atom_order`。

描写/空间题的 QMAP 必须写清：`domain=描写/空间桥接` 时，主入口、快速表、证据表统一指向 `description_space_bridge_cards`；查建筑结构、室内陈设、景物环境怎样落在某空间，默认只取 `link_strength='strong'`，并按 `link_type` 分层。

人物/描写题的 QMAP 必须写清：`domain=人物/描写桥接` 时，主入口表为 `description_person_bridge_cards`，强连接视图为 `v_person_description_strong_cards`，人物汇总视图为 `v_person_identity_description_summary`；它挂在人物身份中枢主干上，先用人物归一取得 `person_code`，再查容貌、服饰、神情、动作、心理等描写证据；默认取 `link_strength='strong'`，`review/unmatched` 只用于补漏和原文复核。

人物/话语题的 QMAP 必须写清：`domain=人物/话语桥接` 时，主入口表为 `v_person_speech_utterance_cards`，快速汇总表为 `v_person_identity_speech_summary`，对话对象表为 `v_person_speech_partner_summary`；它挂在人物身份中枢主干上，先用人物归一取得 `person_code`，再查这个人说过哪些话、对谁说、怎么说、话里有什么。若题目中心是话语距离、邻域或两人对谈密度，则仍走 `domain=话语对话`。

人物/命运关系题的 QMAP 必须写清：`domain=人物/命运关系桥接` 时，主入口表为 `v_person_fate_relation_cards`，快速汇总表为 `v_person_identity_fate_relation_summary`；它挂在人物身份中枢主干上，先用人物归一取得 `person_code`，再查这个人的个人命运节点、参与的两两交集、参与的三人交集、参与的关系弧线。若题目中心是宝黛钗交集本身，可再回 `pair_fate_convergence_* / trio_fate_convergence_*`；若题目中心是宝黛关系弧线本身，可再回 `baodai_relationship_arc_points / baodai_story_time_alignment`。人物中心问法先按人查。

人物/空间题的 QMAP 必须写清：`domain=人物/空间桥接` 时，主入口表为 `v_person_space_cards`，快速汇总表为 `v_person_identity_space_summary`；它挂在人物身份中枢主干上，先用人物归一取得 `person_code`，再按 `card_type` 区分空间归属、空间主场、同聚点空间出现、行动路线主体、路线空间归属。问“宝玉有没有去过潇湘馆”“某人住哪里”“某人的路线经过哪里”，都先走这里。

人物/事件题的 QMAP 必须写清：`domain=人物/事件桥接` 时，主入口表为 `v_person_event_cards`，快速汇总表为 `v_person_identity_event_summary`；它挂在人物身份中枢主干上，先用人物归一取得 `person_code`，再按 `event_id` 接故事推进节点和原文。事件参与不等于事件主因，必须回 `event_relation_type / person_role / atom_code` 裁判。

人物认知中枢题的 QMAP 必须写清：`domain=人物认知中枢` 时，主入口和快速表为 `v_person_cognitive_hub_dimension_summary`；样板表为 `v_baoyu_cognitive_hub_sample`。凡问“某人的中心页、某人的全景、人物全维交集”，先从这张总览看哪些维度已成型，再转入描写、话语、关系、命运、空间、事件等具体入口。

## 现场交互修正活账硬锁

坐标工程允许人机中途交流，但交流不是另开聊天线，而是一次状态回流。触发词不是死按钮，也不是封闭词表；它只是帮助 AI 看清当前讨论焦点。只要用户中途围绕“子问题 / 拆分 / 取词 / 词网 / 策略 / 方法 / 模板 / 执行顺序 / 补证 / 材料不够 / 四态 / 放行 / 重跑”等内容讨论，随后说“好的 / 同意 / 继续 / 继续修正 / 继续完善 / 进入工程再执行 / 再跑 / 重跑”等放行话，AI 不得直接继续往下跑。

必须先写或续写 `Codex现场修正活账`，并按前面真实讨论的焦点判定本轮回到哪个策略讨论节点：

```text
子问题策略讨论节点：子问题、拆分、拆题、两两对象、两两交集、问题树、问题拆成几问。
取词-查表策略讨论节点：取词、词网、强复合、词角色、策略、方法、模板、查证顺序、库线优先级、用表收点。
补证诊断门：材料不够、证据不够、补证、补查、内容不足、需要重跑材料。
四态放行诊断门：材料池、四态、放行、入池、精读材料词、写作前追证、最终答案。
```

其中补证诊断门和四态放行诊断门不是新根门；它们发现问题后，必须回流到子问题策略讨论节点或取词-查表策略讨论节点。若只能判断“材料不够”但说不清是拆分漏了还是取词查表策略错了，状态只能写“阻断待判”，不能继续写最终答案。

## 人工交互后原位置续跑硬锁

人工交互不是重新开题，也不是覆盖旧流程；它只能把当前题回送到已经存在的策略组原位置。用户确认、纠偏、补充、放行以后，机器必须先判断“回到哪里”，再从当前激活版本继续跑。

| 交互焦点 | 回到的原位置 | 必须续写的账 | 续跑边界 |
|---|---|---|---|
| 子问题、拆题、两两交集、层级规则 | `000E_B` 模板00子问题策略讨论节点 | `SQ-v__`、子问题状态、两两交集表、可查性裁判 | 只改受影响子问题；未确认子问题不得喂下一步 |
| 查询词、词网、强复合、查表方法、模板选择 | `000E_B` 取词查表与成场策略组及 01-11 模板 | `TS-v__`、保留词、增补词、删减词、重跑范围 | 只重跑受影响查库路径 |
| 原文证据不足、候选噪声、同场/距离不稳 | 原文裁判与入池策略组 | `EV-v__`、原文窗口、证据等级、补证范围 | 不改题意，先补原文裁判 |
| 材料池不满、四态不稳、材料词缺失 | 原文裁判与入池策略组；必要时回到拆题或取词 | `MAT-v__`、材料池四态、材料词归账、补查范围 | 不写最终答案，先补材料 |
| 用户要求总结、复盘、继续完善 | 交互修正与交付复盘策略组；若发现缺口，回到上面对应组 | `ANS-v__` 或当前激活版本综合账 | 只从当前激活版本和可保留旧材料综合 |

硬规则：每一次人工交互后，都必须写明“本轮回到的原位置”。不能只写“继续推进”；不能把纠偏后的新口径直接接到后面跑；不能跳过短账、活账、子问题状态和材料词归账。

## 两个策略讨论节点

入口这里只做指路，不另造模板。真正有效位置在 `000E_B`：

```text
子问题策略讨论节点：进入 000E_B 的“模板 00｜坐标工程复杂题拆子问题”。这里用两两交集来设计子问题。
取词-查表策略讨论节点：进入 000E_B 的具体 01-11 策略模板和结构转化表。这里用两两交集来查表收点。
```

两两交集有两次用法：前端拆题时，用它把复杂问题拆成两两可查子问题；后端查询时，用它把两个编码、两个变量或一个人物加一个物/事/文类/场景交叉成坐标点。用户说“现在讨论子问题”，AI 必须回到模板00显示已有 `Codex子问题拆解策略`、`Codex子问题`、`Codex子问题两两交集表` 和可查性裁判；没有写入就说未写入，不能临时编。用户说“现在讨论策略 / 取词 / 方法”，AI 必须显示已有取词-查表策略快照；没有写入就说未写入，不能临时编。

现场修正活账必须至少保留这些字段：

```text
Codex修正触发：用户原话、讨论焦点或放行话。
Codex修正归类：子问题策略讨论节点 / 取词-查表策略讨论节点 / 补证诊断门 / 四态放行诊断门。
Codex回到门：本轮真正回到哪里继续。
Codex保留旧内容：前轮哪些子问题、查询词、策略、材料仍可用。
Codex新增内容：本轮新增什么。
Codex删减内容：本轮废弃或降级什么。
Codex重跑范围：只重跑哪一段，不许无故全盘失忆。
Codex当前激活版本：SQ-v__ / TS-v__ / EV-v__ / MAT-v__ / ANS-v__。
Codex最终综合依据：最终答案只从当前激活版本 + 标记可保留的旧材料综合。
```

工程硬结论：修正不是覆盖，是追加 delta；旧内容必须被标为“保留使用、部分保留、已被替代、错误废弃、待复核、转为背景材料”之一。没有现场修正活账，材料门只能阻断待补；没有当前激活版本和最终综合依据，不能进入红楼解语。

## 策略随身短账硬锁

现场修正活账管“用户纠偏以后回到哪里”；策略随身短账管“每一步是否真的完成”。两者不是二选一：有纠偏时必须写活账；每个策略组推进时必须写短账。

短账必须保持轻量，不写长文，但必须足够复盘。最低字段：

```text
Codex短账编号：R__ / SQ__ / TS__ / EV__ / MAT__。
Codex策略组：入门拆题 / 取词查表 / 原文裁判 / 入池四态 / 交互修正 / 交付复盘。
Codex当前子问题：正在回答哪一条；若无子问题，说明为什么不拆题。
Codex本轮规则：本轮入围、排除、放宽、降级口径。
Codex查询词与编码：本轮使用的词、强复合、查询编码或入口表。
Codex查库入口：唯一中心大库中的入口表或视图。
Codex原文裁判：原文支持 / 原文不支持 / 待补证。
Codex子问题材料词归账：本子问题留下了哪些材料词、原文锚点和材料池入口候选。
Codex满意度与补查：满意 / 不满意；若不满意，二查或三查的范围是什么。
Codex结果状态：待查 / 已回答 / 已修改 / 已确认 / 可喂下一步 / 只作弱证 / 不可喂下一步。
Codex下一步：继续查 / 二查 / 三查 / 回退拆题 / 回退取词 / 入池。
```

短账的作用不是给用户看热闹，而是给下一步机器自己读。下一轮进入同一题、同一策略组或用户要求复盘时，必须先读本轮短账和当前激活版本，再继续执行。

## 取词归一硬规则

坐标工程不靠机器硬切自然句，也不让取词抢在子问题和主中枢路径之前。

进入坐标查询以后，必须先读总脑、拆子问题、选主中枢与路径、生成 QMAP，再取词归一、查表、回原文、入材料池。

```text
自然问题
  -> 拆子问题
  -> 选主中枢与路径
  -> QMAP
  -> 取词归一
  -> Codex查询词 / Codex强复合 / Codex词角色 / Codex材料升级条件
  -> Codex查询逻辑策略
  -> Codex查证顺序
  -> formal_honglou_coordinate_material_gate.material_pack_bundle()
  -> 材料池入口候选
```

硬规则：

```text
人物轴：人名不拆、不猜；必须先接 70/72 全人物归一包；变量投影里人物只按 variable_value_name 精确命中。
变量轴：先查 variable_value_name 精确命中；没有精确命中时，二字以上词才允许包含补救。
词位轴：明确正文词 / 短语 / 单字，先走全文词位库穷尽。
单字轴：单字分三层处理，不能一刀切。
  1. 物象根字可以保留：如竹可以引出竹子、竹叶、竹梢；金可以引出黄金、金器；花、泪、黄等可作物象/颜色/意象根轴。
  2. 明确问字词时可以保留：如“黄字”“金这个字”“眼泪的泪”。
  3. 人名拆字和宽网附带字不得自动升主查：如“黛玉”的“玉”、由“泪”宽泛带出的“珠”，只能作补查提示，不能冒充主轴。
代词轴：我、你、他、她、自己、本人不得作为主查词；它们只能作为说话边界、叙述视角或裁判条件。
同场轴：same_atom / same_scene / same_event 只表示候选接近，不等于最终结论。
原文轴：每个候选必须带 atom_code、segment_no、chapter_no、source_sentence。
```

## 坐标查询取词心法

坐标门不是裸搜工具。坐标工程有自己的八步思考和 Codex 查询词路：

```text
AI 先定中心。
AI 再定路径。
AI 再定词网。
AI 再定执行顺序。
工具只按这些令牌查库，不替 AI 猜词。
```

坐标门的 AI 取词必须完成六件事：

```text
1. 定中心：本题到底是人物、物象、空间、季节、事件、原句、距离、同场，还是多轴交叉。
2. 定主查词：只放原问题必须成立的核心对象。
3. 定强复合：把必须共同成立的对象放进 compound_groups，防止单边材料冒充关系证据。
4. 定词角色：区分主查词、归一词、扩展词、背景锚点、排除词。
5. 定查询词策略：说明中心词为什么入网，扩展词、背景锚点、排除词为什么保留或降级。
6. 定升级条件：什么材料可以入材料池，什么只能算候选或补查入口。
```

定完这六件事后，必须进入坐标工程查询逻辑策略经验模板组选择执行策略。这里不是“查询词策略”，而是取词之后的查询逻辑策略经验层：

```text
000E_B_坐标工程查询逻辑策略经验模板组_新窗口学习入口.md
```

也就是：先 AI 取词，再定中心、定路径、定词网，最后在坐标工程查询逻辑策略经验模板组里定执行顺序。人物题、人物+人物题、人物+物象交集题、贡献题、忌讳题、诗词文题等，都应先落入对应查询逻辑策略经验模板，再生成 Codex查证顺序。

强制字段：

```text
Codex查询词策略：中心词=____；强复合依据=____；扩展词=____；背景锚点=____；排除词=____；取舍理由=____
Codex查询逻辑策略：模板编号｜模板名称｜为什么选它
Codex结构转化表：题目中心=____；选择结构=人物结构/描写结构/推进结构/证据结构；查询编码=____；交叉办法=____；选择理由=____；下一步查证动作=____
```

如果所有策略都不合适，必须写：

```text
Codex查询逻辑策略：模板00｜拆子问题策略
Codex子问题：
1. ____
2. ____
Codex子问题策略队列：
1. 子问题：____｜两两可查：对象A + 对象B｜重新选择策略：模板__｜Codex查询词策略：____｜Codex查询词：____｜Codex强复合：____｜Codex查证顺序：____｜Codex材料升级条件：____
2. 子问题：____｜两两可查：对象A + 对象B｜重新选择策略：模板__｜Codex查询词策略：____｜Codex查询词：____｜Codex强复合：____｜Codex查证顺序：____｜Codex材料升级条件：____
```

模板00的子问题不能只列标题。子问题产生后必须分别排队，每个子问题重新进入 `000E_B` 的坐标工程查询逻辑策略经验模板选择；四个子问题就是四次策略选择。每条子问题必须写成两两可查的对象或强复合，再生成自己的查证顺序。

词的身份不可混：

```text
主查词：本题真正要证明关系的核心对象，进入 Codex查询词 和 Codex强复合。
归一词：人物别名、简称、繁简、异写；交给 70/72 人物归一或正字归一，不当第一轮执行词。
扩展词：同类物象、动作、状态、场景词，只用于补漏，不能自动升主证。
背景锚点：地点、回目、标题、名场面、诗词入口，只帮助定位，不能单独冒充答案。
排除词：容易串题、同字误召回、旧题污染、标题诱导和无关热词，用于降级或补查。
查询词策略：把以上取舍写成可复盘理由；只写词不写策略，等于没有证明词网为什么这样形成。
```

坐标门必须执行下列对象判断：

```text
人物对象：人名不拆、不猜。人物组只放归一人物实体，例如林黛玉；黛玉、林姑娘、颦儿等放归一词，由人物包处理。
物象对象：可抓最小有效物象字/词。竹可以作为竹子、竹叶、竹梢、竹影等物象根；金、玉、泪、帕、灯等单字物象也可以作主查词。
复合物对象：必须守住本体。竹扇可提示扇和竹两线，但普通扇不能自动算竹，必须靠原文或变量证据升级。
关系题：第一组通常是人物/主体，第二组是物象/空间/事件/季节；动作、情绪、地点、回目多为辅助，不轻易进强复合。
补查题：首轮单边、标题化、摘要化、材料互不支撑时，回到 AI 取词层调整词角色和强复合，不让程序自动扩大。
```

## 诗词文题取材专门规则

第一取材入口应优先走：

```text

```

执行顺序：

```text

2. 若题目带人物，再叠加 target_people / 人物变量。
3. 若题目带场景，再叠加 segment_no / scene / space / event 相关锚点。
```

```text

```

降级条件：

```text

同一文本可能被重复分类，如曲 / 红楼梦曲、判词 / 七绝 / 诗谶。

```


## 自述梦裁判门

遇到“某人自己说梦 / 亲口说做梦 / 醒后讲梦 / 自述梦”这类题，不新增大库，只加裁判门。

```text

04_坐标入材料池凭证门.json
```

这两个文件是坐标工程自己的入材料池凭证门。

它们必须写清：

```text
material_admission_status：准入待判 / 阻断待补
material_admission_reason：为什么能交材料池，或为什么阻断
material_admission_rule：坐标候选必须具备坐标取材来源、入口词路、原子段/回目/原文句锚点
codex_query_lane：coordinate
source_system：坐标取材包
source_trace：坐标查询中心库（variable_points / v_variable_point_context）/ 词位库 / 字位桥表 / 距离共场
atom_code / segment_no / chapter_no / source_sentence
evidence_layer / final_answer_status
material_term_hits / material_coordinate_hits / material_nearest_pairs / material_cooccurrence
```

如果 `04_坐标入材料池凭证门` 缺失，状态不能写“已入材料池”，只能写：

```text
坐标取材已命中，待补坐标入材料池凭证门。
```

如果凭证门状态是 `阻断待补`，材料不能进入红楼解语，只能进入补证。

## 坐标工程验收矩阵

坐标工程成功与否，只用坐标工程自己的字段验前半段。

必须用下表验收：

| 验收项 | 通过标准 |
|---|---|
| 坐标触发 | 出现 `取材门：坐标` 和 `recall_gate：coordinate`。 |
| 查询词路 | 出现 `Codex查询词策略`、`Codex查询词`、`Codex强复合`、`Codex词角色`。 |
| 结构转化表 | 出现 `Codex结构转化表`，能回答选择结构、查询编码、选择理由、下一步查证动作。 |
| 000E_B坐标工程策略经验 | 出现 `Codex查询逻辑策略` 和 `Codex结构转化表`；模板00题必须出现 `Codex子问题策略队列`。 |
| 人物归一 | 人物题出现 `character_key` 和归一正名。 |
| 坐标预检 | `coordinate_preflight_status=通过`，或明确写出需补证令牌。 |
| 词位取材 | 有 `material_term_hits`，或说明本题没有明确字词入口。 |
| 变量取材 | 有 `material_coordinate_hits`，或说明本题没有变量入口。 |
| 距离 / 共场 | 最近、同场、同时、共同出现题必须有 `material_nearest_pairs` 或 `material_cooccurrence`。 |
| 原子段 | 有 `atom_code`、`segment_no`、`global_atom_order` 中的可用字段。 |
| 原文句 | 有 `material_source_sentences` 或等价原文句。 |
| 材料池入口 | 有 `04_坐标入材料池凭证门.md/json`，且状态为 `准入待判` 或明确阻断原因。 |
| 写作阻断 | 未完成材料池四态、精读材料词、写作前追证前，不写最终答案。 |

如果缺少坐标触发、查询词路、人物归一强令牌或坐标预检，状态只能写：

```text
坐标工程阻断：未完成坐标取材入口。
```

如果已有坐标材料候选但尚未送入材料池入口，状态只能写：

```text
坐标取材已命中，待入材料池。
```

如果已送入材料池入口但后半身未完成，状态只能写：

```text
坐标材料已入池，待材料池四态 / 精读材料词 / 写作前追证。
```

## 状态词

坐标工程只能这样报告状态：

```text
已触发坐标工程 = 已识别坐标法，并生成取材门：坐标。
已生成坐标取材包 = 已通过坐标预检并生成坐标候选与坐标入池凭证。
已完成坐标取材 = 已提出原子段、原文句、变量点或词位点。
已送入材料池入口 = 坐标候选已进入总证据池 / 材料池入口。
不可写最终答案 = 未完成材料池四态、精读材料词、写作前原文追证。
可写最终答案 = 已完成 SQ完成卡、材料池四态、精读材料词、写作前原文追证，并且原文证据可支撑作答。
```

如果用户说：

```text
进入红楼梦工程，用坐标法查询……
```

新线程必须理解成：

```text
进入坐标查询。
先走坐标工程。
不得离开坐标取材门。
坐标取材完成后，再把材料送入材料池入口。
```

```text
进入坐标查询。
我要查：……
```
