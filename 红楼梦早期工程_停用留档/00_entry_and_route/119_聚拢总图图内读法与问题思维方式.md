## 一句话定案

AI 进入聚拢总图以后，不是看见标签就写答案，也不是看见命中就写答案。

它必须先问：

```text
我现在抓到的是什么节点？
这个问题要我在图里做什么动作？
这些标签是入口、候选、证据，还是质量提示？
我应该放大、缩小、交集，还是串域？
最后能不能回原文裁判？
这些命中有没有完成聚拢裁判？
```

## 一、为什么要写这个文件

116 已经封死旧前门。
117 已经写了新入口和触发词。

但还缺一件事：

```text
AI 进了聚拢总图以后，面对 nodes / tags / edges / closure / line_membership，它到底怎么想？
```

如果没有这个读法，AI 可能会犯新错误：

```text
把标签当答案。
把候选当强证。
把同图邻近当同场。
把 101 场候选当原文裁判。
把聚拢域当已经证明的结论。
```

所以必须给它图内思维方式。

## 二、进图后的第一眼：先判节点身份

AI 抓到任何节点，第一眼必须看：

```text
node_id
node_type
node_type_cn
title
summary
source_ref
quality_status
evidence_status
source_quote
```

先判断它是哪一种节点：

```text
line_index：一行索引，最接近原文。
atom_segment：原子段，最小证据圈。
aggregate_segment：聚拢段，小语境。
aggregate_unit：聚拢单元，小动作 / 小对话 / 小转折。
aggregate_event：聚拢事件，一件事。
aggregate_scene：聚拢场候选，一个现场入口。
aggregate_domain：聚拢域候选，多个场的高层分析空间。
```

不同节点不能同等使用。

## 三、标签不是答案

标签只回答：

```text
这里可能涉及什么。
```

标签不回答：

```text
最终结论是什么。
```

标签分四类看。

### 1. 入口标签

用于把问题带进图。

例子：

```text
人物：林黛玉
物象：宫花
空间：潇湘馆
动作：送物
主题：冷
```

作用：

```text
找候选节点。
```

### 2. 语境标签

用于理解节点周围是什么。

例子：

```text
事件类型：探病 / 宴饮 / 梦境 / 诗社
场点性质：focus_scene / functional_scene / passing_scene
在场模式：real_presence / mentioned_only / memory_or_recall
```

作用：

```text
判断这个节点该怎么用。
```

### 3. 证据质量标签

用于判断能不能入材料池。

例子：

```text
matched
weak_match
needs_review
line_id_pending
evidence_eligible
```

作用：

```text
决定是否需要回原文、补证、降级使用。
```

### 4. 分析提示标签

用于引导高层解释，但不能直接当结论。

例子：

```text
命运提示
关系提示
主题提示
文本功能
象征提示
```

作用：

```text
提示可以串聚拢域。
```

## 四、问题决定盘法

AI 不是进图后乱盘。

问题类型决定图内动作。

### 1. 精确事实题

例子：

```text
第六回回目是什么？
某一句原文在哪里？
```

图内动作：

```text
入口线索 -> 找最小节点 -> 缩小到一行 / 原文 -> 直接回原文裁判
```

原则：

```text
能小就小，不必上升到场或域。
```

### 2. 同场题

例子：

```text
妙玉和林黛玉有没有同场？
```

图内动作：

```text
人物标签找两组节点
-> 放大到聚拢场候选
-> 求交集
-> 缩小到事件 / 单元 / 原文
-> 判在场模式
```

必须区分：

```text
真实同场
同场无互动
同回不同场
被提及不在场
回忆 / 梦境 / 远端影响
证据不足
```

### 3. 关系题

例子：

```text
宝玉和贾政关系如何变化？
```

图内动作：

```text
两人物入口标签
-> 找共同节点 / 关系边
-> 放大到事件和场
-> 按时间排序
-> 串关系域
-> 回原文裁判
```

必须输出：

```text
关系发生点
关系变化点
关键场
证据强弱
解释边界
```

### 4. 物件流转题

例子：

```text
宫花是谁拿出来送人的？
```

图内动作：

```text
物象标签找点
-> 动作标签交集
-> 放大到事件 / 场
-> 按传递顺序串流转域
-> 回原文裁判
```

必须输出：

```text
谁拿出
谁传递
谁接收
在哪个场
说了什么
物件功能是什么
```

### 5. 命运 / 病程 / 发展题

例子：

```text
林黛玉病情怎么发展？
```

图内动作：

```text
人物标签 + 病/药/医治标签
-> 找相关节点
-> 放大到场
-> 按时间排序
-> 串病程域 / 命运域
-> 回原文裁判
```

必须输出：

```text
起点
推进
转折
加重
收束
原文证据
```

### 6. 主题 / 深度解释题

例子：

```text
为什么说红楼梦很冷？
```

图内动作：

```text
主题词入口
-> 多类标签会合
-> 多个场形成聚拢域
-> 分层材料池
-> 回原文裁判
-> 谨慎解释
```

原则：

```text
主题必须落到场、事件、原文编号，不能空泛抒情。
```

## 五、三问法：每个候选节点都要问

AI 看到一个候选节点，必须问三句话：

```text
它能不能直接回答问题？
它需要向上放大看场 / 域吗？
它需要向下缩小回原文吗？
```

### 能直接回答

条件：

```text
问题很小。
节点接近原文。
source_quote 清楚。
不涉及复杂关系、同场、发展。
```

### 需要向上放大

条件：

```text
单点不够解释。
需要看同一单元、事件、场。
问题涉及关系、同场、动作、物件流转。
```

### 需要向下缩小

条件：

```text
节点太大。
标签太泛。
候选场需要裁判。
要判断真实在场或原文是否支持。
```

## 六、图内读法顺序

标准图内读法：

```text
1. 看入口线索。
2. 找候选节点。
3. 看节点类型。
4. 看标签类型。
5. 看质量状态。
6. 决定盘法：放大 / 缩小 / 交集 / 串域。
7. 回原文裁判。
8. 进入材料池。
9. 材料池四态判定。
10. 精读材料词 / 聚拢裁判单。
11. 写作前原文追证摘抄。
12. 红楼解语。
```

## 七、不能做的事

```text
不能看见人物标签就说这个人在场。
不能看见关系提示就说关系成立。
不能看见 101 presence_mode 就直接当强证。
不能看见主题标签就写深度结论。
不能把同回当同场。
不能把同场当互动。
不能把被提及当亲自在场。
不能把聚拢域候选当最终命运解释。
```

## 八、聚拢域怎么用

聚拢域是场以上的分析空间。

它不是固定大圈，而是从多个场里抽取同类标签形成的高层场域。

常见聚拢域：

```text
人物命运域
关系域
病程域
物件流转域
空间路线域
主题域
文本功能域
梦域
```

使用规则：

```text
先有多个场。
再看这些场是否有同类标签。
再按问题意图串域。
最后回到每个场和原文编号。
```

聚拢域不能直接写答案。

聚拢域只负责：

```text
组织高层材料。
```

## 九、和材料池的关系

材料池不是把图里的东西全搬进去。

材料池只收经过判断的可用材料。

材料池条目必须写：

```text
入口线索
候选节点
节点类型
图内动作
原文编号
source_quote
证据强弱
是否回原文裁判
回答哪个子问题
是否需要补证
```

## 十、最终一句话

```text
进图以后，先辨节点，再读标签，再选盘法，再回原文。
```

再短一点：

```text
标签给方向。
节点给位置。
边给关系。
归属链给上下文。
原文给裁判。
```
