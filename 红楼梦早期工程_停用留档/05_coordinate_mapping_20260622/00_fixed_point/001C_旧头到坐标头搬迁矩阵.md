# 001C_旧头到坐标头搬迁矩阵

生成时间：2026-06-21

用途：把旧红楼梦工程头部逐项对应到坐标头，保证完整性，不漏旧规则，也不把旧取材词误搬到坐标头。

原则：

```text
同一个身子，同一个尾巴，两个头。
旧头保留：进入红楼梦工程。
坐标头新增：进入红楼梦人工智能咨询工程。
坐标头只替换第一取材入口，不替换材料池以后流程。
```

## 三、哪些不能搬到坐标头前半段

这些旧头内容不能放在坐标头第一动作里：

| 禁搬项 | 原因 | 坐标头替代 |
|---|---|---|
| 先建本题闭环包 | 会把坐标题吸回旧路线 | 先坐标分流令牌。 |
| 先进入聚拢库取材单 | 会抢在坐标门之前 | 先坐标材料门。 |
| 先跑旧搜索词网络 | 会变成语义/全文补漏路线 | 先词位库 + 坐标变量库。 |
| 先自由查 formal_honglou_search.sqlite | 旧头也禁止，自然不能搬 | 先坐标三库取材。 |
| 先按旧语义召回 | A/B 分流会失败 | 取材门：坐标。 |
| 用聚拢四件套证明坐标取材已进门 | 坐标前半段不是旧闭环包 | 用 coordinate_preflight、material_term_hits、material_coordinate_hits、material_source_atoms 证明。 |

## 四、哪些必须完整继承

这些旧头内容必须完整继承，不能因为坐标头而丢掉：

| 必继承项 | 坐标头写法 |
|---|---|
| 不自由搜索 | 坐标头也不自由搜索，必须走坐标材料门。 |
| 不凭印象回答 | 坐标头只取材，不裸答。 |
| 命中不是答案 | 坐标命中也只是候选。 |
| 人物先归一 | 70/72 全人物归一包是强令牌。 |
| 查询词不是答案 | Codex查询词只服务取材。 |
| 原文负责裁判 | 坐标库定位后必须回原文句。 |
| 材料必须入池 | 坐标候选必须送入同一个材料池入口。 |
| 材料池四态 | 坐标候选仍要判可用 / 背景 / 不可用 / 需补证。 |
| 精读材料词 | 材料池之后仍需精读材料词。 |
| 写作前追证 | 最终写作前仍需原文追证摘抄。 |

## 四-A、漏搬补签矩阵：旧聚拢头“取词心法”逐条搬入坐标头

这一节专门补上前一版矩阵漏得不够细的部分。

旧聚拢头不是只有“查询词规则”四个字，它有一整套 Codex 取词前置心法。坐标头必须完整继承这套心法，但执行后端从“聚拢 / 语义 / 闭环包”改成“词位库 / 变量投影库 / 字位桥表 / 距离共场”。

| 旧聚拢头真实心法 | 旧头原作用 | 坐标头搬入后的写法 | 改名 / 禁用 | 验收方式 |
|---|---|---|---|---|
| 定中心 | 先判断题目到底问人物、关系、物象、空间、诗词、事件、主题、定位，还是多轴交叉 | 坐标头第一步也必须先定问题中心，写成 `Codex问题中心` | 不改原则，只把后续取材入口改坐标 | 入口词路里必须出现问题中心，不能直接查库 |
| 定路径 | 从经验仓和库线骨架选择查证路线 | 坐标头写成 `Codex经验路径`、`Codex库线路径`，但库线要落到词位、变量、字位、距离、共场 | 禁止把路径写成“先建闭环包” | 词路里必须说明为什么走词位 / 坐标 / 距离 / 共场 |
| 定词网 | 生成 search_terms、compound_groups、anchor_terms | 坐标头写成 `Codex查询词`、`Codex强复合`、`Codex背景锚点` | 不让机器编译器硬切；由 AI 取词 | 缺 `Codex查询词` 不许进坐标材料门 |
| 定执行 | 给 preferred_libraries、source_order、guardrails、next_action_directive | 坐标头写成 `Codex优先库`、`Codex查证顺序`、`Codex注意事项`、`Codex下一步指令` | preferred_libraries 中禁用旧闭环包作第一入口 | 查证顺序必须能落到坐标三库和原文回查 |
| 定资料化问题 | 拆的是资料需求，不是观点提纲 | 坐标头也只拆“要查哪些变量、哪些词位、哪些原子段、哪些原文句” | 禁止提前写观点结论 | 输出必须是材料池候选，不是文章 |
| 策略质量自检 | 检查路径、对象、复合轴、背景锚点、下一步动作是否清楚 | 坐标头搬成坐标预检：对象是否齐、人物是否归一、词位/变量/距离/共场是否该走 | 自检不等于答案裁判 | `coordinate_preflight` 通过或明确需补证 |
| 主查词 | 真正要证明关系的核心对象 | 坐标头写入 `Codex查询词` 和必要的 `Codex强复合` | 人物主查词只放归一人物实体；不放别名堆 | 查询词包里人物是归一实体，物象可为最小有效根字 |
| 归一词 | 人物别名、简称、繁简体、异写，用来防漏 | 坐标头写入 `Codex词角色：归一词=...`，并交给人物归一包或文字折叠处理 | 归一词不当第一轮主查词 | 人物题必须显示已走 70/72 全人物归一包 |
| 扩展词 | 同类物象、动作、状态、场景词，用来扩大候选 | 坐标头写入 `Codex词角色：扩展词=...` | 扩展词不能自动升主证 | 扩展命中只能作补查或旁证，需原文升级 |
| 背景锚点 | 地点、回目、标题、名场面、诗词入口，用来定位场景 | 坐标头写入 `Codex背景锚点` / `anchor_terms` | 背景锚点不能冒充答案 | 同回 / 同标题只能降权，不算同场 |
| 反证 / 排除词 | 排除串题、同字误召回、标题诱导和旧题污染 | 坐标头写入 `Codex注意事项` 或 `排除词` | 不作为正向查证主轴 | 材料包 warnings 记录误召回风险 |

## 四-B、漏搬补签矩阵：旧聚拢头“对象取词规则”逐条搬入坐标头

这一节解决用户特别指出的两类问题：人名不能拆，物象单字不能乱禁。

| 对象类型 | 旧聚拢头成熟规则 | 坐标头必须继承的规则 | 坐标头执行位置 | 禁止误搬 |
|---|---|---|---|---|
| 人物 / 人名 | 人物对象先入人物库归一；执行词只放归一人物实体 | 人名不拆、不猜、不按字取词；只走 70/72 全人物归一包 | AI 取词后、坐标材料门前，作为最强令牌 | 禁止把“林黛玉”拆成“林 / 黛 / 玉”；禁止让别名堆进第一轮主查词 |
| 人物别名 / 称谓 | 黛玉、林姑娘、颦儿、潇湘妃子等放归一词 | 作为 normalization_terms，交给人物归一，不作为全文硬扫主轴 | `Codex词角色：归一词` | 禁止用别名列表替代人物归一包 |
| 物象 / 器物 / 花木 | 先抓最小有效物象字 / 词，再抓复合词、别称、器物形态、场景形态 | 单字物象可以是强主查词：竹、金、玉、泪、帕、灯、花、黄等，前提是题目语义就是这个物象 / 字 / 颜色 / 意象 | AI 取词阶段决定，坐标材料门只验词和查表 | 禁止按“短单字一律降级”处理 |
| 物象复合 | 竹子题可用竹、竹子、竹枝、竹叶、竹影、竹梢、竹帘、竹扇 | 坐标头先精确查变量名，再用词位穷尽防漏，再回原文裁判 | 变量投影库 + 全文词位库 + 原文句 | 禁止 `raw_value LIKE` 泛化结果直接冒充主证 |
| 复合本体 | 竹扇可以提示扇类和竹类，但普通扇不能自动算竹 | 复合物要守住本体，只有原文或上下文确认关系才升级 | 原文裁判 / 材料池四态 | 禁止由同类词自动反推材质或象征 |
| 动作 / 情绪 | 可作辅助线索和扩展词 | 写入扩展词或背景锚点，不进入强复合资格，除非题目核心就是动作 / 情绪关系 | AI 取词阶段 | 禁止把“哭、住、题、泪”等辅助词和人物并列成强主证，除非问题明确问这些词 |
| 空间 / 回目 / 标题 | 背景锚点，用来定位场景 | 坐标头可查 space / chapter / title，但只能帮助组证 | 变量投影库 / 字位桥表 / 原文回查 | 禁止同回当同场，禁止标题当正文证据 |

## 四-C、漏搬补签矩阵：旧聚拢头“库线认识顺序”改造成坐标库线

| 旧聚拢头认识顺序 | 旧头执行方式 | 坐标头执行方式 | 是否继承 | 坐标头硬令 |
|---|---|---|---|---|
| 先判断知识对象 | 判断人、物、空间、诗词、事件、主题、原文位置 | 同样先判断对象类型，再决定词位 / 变量 / 距离 / 共场 | 完整继承 | 不能没判断对象就进表 |
| 先盘库，再回原文，再建材料池 | 聚拢库召回后回原文 | 词位库和变量投影库召回后回原文，再入材料池 | 完整继承 | 坐标库只定位，不裁判 |
| 判断关系强度 | 单对象、二元关系、三元共现、跨回呼应、因果链、比较链 | same_atom、same_scene、same_event、same_chapter、directed_distance、cooccurrence | 改工具 | 最近 / 靠近 / 同场题必须出距离或共场候选 |
| 选择库线 | 对象明确先库轴，入口不明先全文词，原句明确先章节真源 | 明确字词先词位库；明确人物/空间/季节/物象/事件先变量库；明确原文位置走字位桥表 | 改库名 | 不让语义召回抢第一入口 |
| 规定材料升级条件 | 只有能回答原问题的原文材料进池 | 坐标候选必须有 atom_code / segment_no / source_sentence，并说明距离 / 共场层级 | 完整继承 | 材料包状态只能到“可入池”，不能直接“可答” |

## 四-D、漏搬补签矩阵：旧聚拢头“二次补查触发”搬入坐标头

| 旧头补查触发 | 坐标头对应补查 | 执行位置 | 降级 / 升级规则 |
|---|---|---|---|
| 首轮只命中单边对象 | 补查另一边核心词和同原子段 / 同场 / 同事件关系 | 坐标材料门后，材料池前 | 单边命中只能作待补证 |
| 首轮只命中标题、地点或摘要 | 回全文词位库和字位桥表找正文句 | 原文回查法 | 标题和地点只作背景锚点 |
| 首轮材料多但互不支撑 | 补查强复合轴、距离、共场、反证词 | 距离 / 共场 / 交集模块 | 多命中不等于关系成立 |
| 首轮材料太少 | 扩大别名、繁简体、物象别称、相关空间或相邻章回 | AI 再给补查词，工具不自动编译 | 扩展词不能直接升主证 |
| 首轮出现旧题内容 | 按本题核心词和 request_id 收缩 | 材料包 warnings / 排除词 | 旧题只能反证或剔除 |
