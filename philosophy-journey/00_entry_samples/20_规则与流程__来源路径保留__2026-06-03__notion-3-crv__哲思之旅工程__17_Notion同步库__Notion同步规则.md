# Notion 同步规则

## 一、同步定位

Notion 是云端写作与临时收件箱。本地“哲思之旅工程”是权威备份、入库、制卡、挂账、复盘和下一课生成的工作区。

## 二、固定收件箱

Notion 数据库：哲思之旅同步台  
数据库链接：https://app.notion.com/p/513267884fbc4224b27feef8b09a03ce  
数据源：collection://98461072-2e74-46fe-b673-2913d555c205

## 三、Notion 字段

| 字段 | 用途 |
|---|---|
| 标题 | 页面标题 |
| 类型 | 课堂记录、课后报告、问题、文章、复盘、底稿、底座、其他 |
| 人物 | 笛卡尔、斯宾诺莎、莱布尼茨、洛克、贝克莱、休谟、康德、通用 |
| 节点 | D01、D02 等 |
| 同步状态 | 待同步、已同步、已入库、需复查 |
| 内容日期 | 内容发生日期 |
| 本地路径 | 本地备份后写入路径 |
| Notion链接 | 原页面链接 |
| 备注 | 特殊说明 |
| 创建时间 | Notion 自动生成 |
| 最后编辑时间 | Notion 自动生成 |

## 四、每日使用方式

1. 在 Notion 中把当天页面放进“哲思之旅同步台”。
2. 将需要 Codex 领取的页面标记为“同步状态 = 待同步”。
3. 回到 Codex，说：“同步 Notion 增量”。
4. Codex 读取待同步页面，备份到本地。
5. Codex 生成每日同步记录。
6. Codex 根据类型进入课程库、问题库、文章库、复盘库或底稿库。
7. 完成后状态更新为“已同步”或“已入库”。

## 五、同步后的本地动作

| Notion 类型 | 本地去向 | 后续动作 |
|---|---|---|
| 课堂记录 | 02_课程库 | 作为全量母本 |
| 课后报告 | 02_课程库 | 作为回收目录 |
| 问题 | 03_问题库 | 生成问题卡 |
| 文章 | 09_文章库 | 登记文章卡 |
| 复盘 | 10_阶段复盘库 | 更新阶段复盘 |
| 底稿 | 15_我的哲学底稿 | 进入本人认可制 |
| 底座 | 01_底座资料库 | 只作为资料，需确认后接入 |

## 六、备用流程

如果 Notion 查询接口不可用或太慢：

1. 你在 Notion 里导出当天页面为 Markdown 或 HTML。
2. 放入 `17_Notion同步库/Notion原文备份/YYYY-MM-DD/`。
3. Codex 直接读本地导出文件。

备用流程不影响工程规则：原话仍一字不改保存，内容仍进入本地库。
