## 三点五、闭环包凭证门

短卡不能只写“已进入红楼梦工程”。

有具体问题时，外显短卡必须带三个机器凭证：

```text
本题闭环包：package_dir=____
聚拢四件套：00AC/00AG/00AI/00AM 已齐 / 缺____
当前状态：已接入入口 / 已建闭环包 / 已进入聚拢库 / 阻断
```

没有 package_dir，或四件套缺任一项，不能说“已进入聚拢库”，只能说：

```text
已接入红楼梦工程入口，正在补本题闭环包。
```

## 四、本题入口词卡

有具体问题时，必须输出：

```text
本题入口词：
对象：____
根字：____
宽网：____
绑定：____
证据源：原文裁判
```

其中：

```text
对象 = 用户真正要查的东西，先守住，不能跑偏。
根字 = 最小字根，如竹、鸡、书、信、药、方、灯。
宽网 = 同义、古典表达、物象规范名、可能称法。
绑定 = 人物、空间、动作、时间、关系等过滤条件。
```

## 五、复杂题才加边界

简单题不需要长篇禁止项。

复杂题、易混题、刚犯过错的题，才加：

```text
本题边界：
不把 ____ 混入 ____。
不把 ____ 当作 ____。
不使用旧专题作为第一证据源。
```

例如林如海荐书题：

```text
不把第4回雨村另写二信混入林如海荐书。
不把第107回恩义反咬当作荐书明文再现。
不使用旧专题里的“吹嘘吹嘘”冒充发现词。
```
