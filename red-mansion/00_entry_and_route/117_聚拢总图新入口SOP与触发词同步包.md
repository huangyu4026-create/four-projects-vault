## 一句话定案

旧前门已经硬封锁。

新前门唯一入口是：

```text
编号递归聚拢总图
```

简称：

```text
聚拢总图
```

运行总法：

```text
问题先抽入口线索。
线索进入聚拢总图。
图内放大、缩小、交集、串域。
最后回原文裁判。
命中只算召回。
聚拢裁判后才允许进入写作。
```

## 一、运行主路

任何正常查询，都必须走：

```text
用户问题
-> 抽入口线索
-> 进入聚拢总图
-> 查 tags / nodes / edges / closure / line_membership
-> 放大 / 缩小 / 交集 / 串域
-> 回原文裁判
-> 入材料池
-> 材料池四态判定
-> 精读材料词 / 聚拢裁判单
-> 写作前原文追证摘抄
-> 写答案
-> 复核
```

禁止回到：

```text
先选库
先查人物库
先查物象库
先查空间库
先走旧八步
先走旧九步
先查 101 场面库
先凭“可直接命中”写红楼解语
```

入口状态从此统一改写：

```text
旧错路：可直接命中 -> 可直接写答案
新：召回已命中 -> 待聚拢裁判
```

任何题都不再按题型开直达口。
简单题走极简聚拢裁判，复杂题走完整聚拢裁判。
没有聚拢裁判单，就没有最终答案。


## 一点五、本题闭环包前置门

任何带具体内容的问题，进入 SOP 后第一件机器动作不是查 sqlite，也不是自由全文搜索，而是创建或确认本题闭环包：

```text
python3 work/formal_honglou_closed_loop.py --question "用户原问题"
```

本题闭环包必须给出 package_dir。

只有 package_dir 内以下四件套齐全并已读，才算进入聚拢库 / 聚拢场：

```text
00AC_聚拢库总入口流程锁.md
00AG_聚拢库取材单.md
00AI_聚拢入材料池清单.csv
00AM_聚拢裁判首读材料池.md
```

缺任一项时，查询只能停在入口修复，不准把全文检索、段落命中或模型判断写成答案。

## 二、入口线索是什么

进入聚拢总图之前，先从问题中抽入口线索。

入口线索包括：

```text
人物
物象
空间
时间
事件
关系
动作
主题词
文本功能词
问题意图
输出要求
```

例子：

```text
妙玉和林黛玉有没有同场？
```

入口线索：

```text
人物：妙玉、林黛玉
问题意图：同场判断
聚拢动作：交集 + 放大到聚拢场 + 回原文裁判
```

例子：

```text
林黛玉的病情怎么发展？
```

入口线索：

```text
人物：林黛玉
物象/动作：病、药、吃药、医治
问题意图：发展过程
聚拢动作：找相关节点 + 串聚拢域 / 病程域 + 回原文裁判
```

例子：

```text
宫花是谁拿出来送人的？
```

入口线索：

```text
物象：宫花
动作：送、分配、流转
问题意图：物件流转路径
聚拢动作：物象标签找点 + 事件/场放大 + 路径串域 + 回原文裁判
```

## 二点五、查询词入口规范：单字宽网与疑义宽网

进入聚拢总图之前，必须先做查询词入口规范化。

核心原则：

```text
查询词不是答案，查询词是撒网。
先单字宽网，再短语收窄，再编号聚拢，再回原文裁判。
```

必须执行：

```text
用户自然词
-> 抽核心对象
-> 拆最小字根
-> 保留题面原词
-> 补常见原文表达
-> 疑义处宁多勿少
-> 进入聚拢总图收点
-> 用人物/空间/动作/时间/关系过滤
-> 回原文裁判
```

最小字根是硬要求：

```text
竹子 -> 竹
书信 / 推荐信 / 荐书 -> 书、信
宫灯 -> 灯
药方 -> 药、方
手帕 -> 帕
活鸡 -> 鸡
```

不得只造一串复合词：

```text
竹板凳、竹衣服、竹扇子、竹画……
```

而忘记最小字根：

```text
竹
```

不得只查：

```text
荐书、妹丈之书
```

而忘记：

```text
书、信
```

疑义词规则：

```text
不确定时宁可多查，不许为了干净而少查。
一个有歧义、有多种写法、有可能被原文改称的对象，20 个、30 个入口词也不算多。
```

但确定专名不乱扩：

```text
林黛玉、贾宝玉、王熙凤这类明确人物，走人物归一和别名表，不临时造 8 个、10 个无根据花词。
```

宽网和收窄分工：

```text
单字负责不漏。
疑义宽网负责多抓可能点。
短语负责收窄。
聚拢总图负责判断。
原文负责裁判。
```

单字宽网只负责收候选点，不允许直接出答案。

材料池必须记录：

```text
题面词
最小字根
短语词
疑义扩展词
过滤条件
命中的编号
为什么保留
为什么排除
是否回原文裁判
```

## 三、触发词规则

### 1. 进入红楼梦工程

触发状态：

```text
进入聚拢总图运行状态。
```

必须执行：

```text
声明不走旧前门。
创建或确认本题闭环包。
自检 00AC/00AG/00AI/00AM。
准备抽入口线索。
准备查聚拢总图。
```

不得执行：

```text
库分级选库。
旧八步/旧九步作为主入口。
101 场面库独立入口。
```

### 2. 展开问题分析思路

触发状态：

```text
只分析入口线索和图内盘法，不正式出答案。
```

必须输出：

```text
入口线索有哪些。
应查聚拢总图的哪些索引。
要做放大、缩小、交集还是串域。
需要回原文裁判的位置。
可能缺哪些补证。
```

### 3. 开始查询

触发状态：

```text
正式进入聚拢总图查证。
```

必须执行：

```text
本题闭环包 -> 00AC/00AG/00AI/00AM 自检 -> 入口线索 -> 聚拢总图候选节点 -> 图内盘法 -> 原文裁判 -> 材料池。
```

### 4. 详查 / 全书 / 穷尽

触发状态：

```text
先查聚拢总图，再决定是否回全文检索补漏。
```

不得一开始直接全文穷尽。

允许回库条件：

```text
聚拢总图无命中。
图内命中不完整。
标签缺失。
映射断裂。
用户明确要求穷尽。
```

### 5. 同场 / 在不在场

触发动作：

```text
对象归一 -> tags 找点 -> 放大到聚拢场 -> 求交集 -> 缩小回原文 -> 判在场模式。
```

输出必须区分：

```text
真实同场
同场无互动
同回不同场
被提及不在场
回忆/梦境/远端影响
证据不足
```

### 6. 关系 / 情感 / 认识变化

触发动作：

```text
对象归一 -> 找共同节点 -> 放大到事件/场 -> 多场串域 -> 回原文裁判。
```

输出必须说明：

```text
关系发生点
关系变化点
关键场
证据强弱
是否只是解释性推论
```

### 7. 命运 / 病程 / 发展

触发动作：

```text
人物或主题标签找点 -> 聚拢场排序 -> 串聚拢域 -> 回原文裁判。
```

输出必须说明：

```text
起点
推进
转折
加重 / 变化
收束
原文证据
```

### 8. 物件 / 流转 / 路径

触发动作：

```text
物象标签找点 -> 动作标签交集 -> 事件/场放大 -> 路径串域 -> 回原文裁判。
```

输出必须说明：

```text
谁拿出
谁传递
谁接收
在哪个场
说了什么
物件功能是什么
```

### 9. 主题 / 为什么 / 深度研究

触发动作：

```text
主题词找点 -> 多类标签会合 -> 多场形成聚拢域 -> 回原文裁判 -> 材料池分层。
```

输出必须说明：

```text
主题不是空泛判断。
必须落到具体场、事件、原文编号。
```

## 四、图内盘法说明

### 放大

```text
一行 -> 原子段 -> 聚拢段 -> 聚拢单元 -> 聚拢事件 -> 聚拢场
```

作用：

```text
一个点不孤立，向上找语境、事件、现场。
```

### 缩小

```text
聚拢场 -> 聚拢事件 -> 聚拢单元 -> 聚拢段 -> 原子段 -> 一行
```

作用：

```text
一个大判断必须落回具体原文。
```

### 交集

```text
对象 A 的节点集合 ∩ 对象 B 的节点集合
对象 A 的节点集合 ∩ 物象/空间/动作节点集合
```

作用：

```text
判断共同语境、共同事件、共同现场。
```

### 串域

```text
多个聚拢场 + 同类标签 -> 聚拢域
```

作用：

```text
回答命运、关系、病程、流转、主题等发展型问题。
```

## 五、聚拢域口径

场以上不再统一叫“弧”。

正式叫：

```text
聚拢域
```

定义：

```text
聚拢域是从多个聚拢场中抽取同类人物、关系、命运、物象、空间、主题或文本功能线索所形成的高层分析场域；它可以包含弧线、路径、节点网和主题群，但必须能回到具体聚拢场和原文编号。
```

例子：

```text
林黛玉命运域
黛玉病程域
宝黛关系域
宫花流转域
冷感主题域
太虚幻境预示域
礼法权力域
梦域
```

## 六、异常回库规则

外部库只在以下情况允许使用：

```text
聚拢总图无命中
标签缺失
映射断裂
needs_review
line_id_pending 需要锚点复核
用户明确要求全文穷尽
用户明确要求审计源库
```

回库以后必须回到聚拢总图：

```text
外部库补点 -> 映射回聚拢总图 -> 回原文裁判 -> 材料池 -> 答案
```

禁止：

```text
外部库 -> 直接答案
```

## 七、材料池新规则

材料池条目必须记录：

```text
入口线索
候选节点
节点类型
所属聚拢场 / 聚拢域候选
原文编号
source_quote
证据等级
是否回原文裁判
回答哪个子问题
是否需要补证
```

材料池不再只写：

```text
来自哪个库
```

而是写：

```text
它在聚拢总图里的位置。
```

## 十、进入红楼梦工程查询准备卡

进入红楼梦工程时，不做长篇宣誓，自动触发轻量查询准备卡。

标准短卡：

```text
已接入红楼梦工程入口。
本题闭环包：已创建 / 未创建。
聚拢四件套：00AC/00AG/00AI/00AM 已齐 / 缺____。
查询准备：先守对象，人物先归一，物象查根字，疑义宽网，编号聚拢，原文裁判。
```

如果用户已经给出问题，必须紧跟本题入口词：

```text
本题入口词：
对象：____
根字：____
宽网：____
绑定：____
证据源：原文裁判
```

复杂题才补“本题边界”，防止把相邻事件、旧专题答案词、结构性后果混成原文直接证据。

详见：`127_进入红楼梦工程查询准备卡触发器.md`
