# 四项目工程｜数字生命 GitHub 试运行包

本包只用于第一阶段试运行：先把“数字生命”一个项目放进 GitHub 骨架，验证目录、规则、同步和版本管理流程。

本阶段不上传四工程全量内容，也不上传大量日记正文、图片、音频、视频或备份压缩包。

## 建议仓库

GitHub 主页：`https://github.com/huangyu4026-create`

建议仓库名：

```text
four-projects-vault
```

如果用户已经建立中文仓库“四项目”，也可以先在该仓库下创建 `digital-life/` 目录作为试运行入口。

## 第一阶段只放什么

1. 数字生命入口说明。
2. 数字生命状态加载与触发词规则。
3. GitHub 同步边界。
4. 收件箱和回包归档目录。
5. 处理报告目录。

## 第一阶段不放什么

1. 不放完整私人日记正文。
2. 不放备份压缩包。
3. 不放大体积图片、音频、视频。
4. 不把 GitHub PR 状态当成数字生命“语义处理完成”的最终裁判。

## 推荐目录

```text
digital-life/
  00_entry/
  01_rules/
  02_style/
  03_triggers/
  04_sync/
  05_inbox/
  06_reports/
  07_assets/
```

## 判断原则

GitHub 用于保存版本、规则和可追溯文件。

Notion 继续作为公共收件箱和人读入口。

本地总账仍然作为“是否真正处理完成”的最终判断依据。
