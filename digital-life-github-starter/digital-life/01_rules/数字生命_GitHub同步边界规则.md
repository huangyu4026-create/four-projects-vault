# 数字生命｜GitHub 同步边界规则

## 1. GitHub 的角色

GitHub 是数字生命的版本正库试运行区，用于保存：

1. 工程规则。
2. 状态加载卡。
3. 触发词表。
4. 同步水位。
5. 收件箱归档。
6. 处理报告。

GitHub 不直接替代本地总账，也不直接替代 Notion 公共收件箱。

## 2. 不上传内容

第一阶段默认不上传：

1. 完整私人日记正文。
2. 备份压缩包。
3. 大量图片、音频、视频。
4. 身份证件、账号、密码、付款信息等敏感信息。
5. 未经用户确认的原始口述全文。

## 3. 可以上传内容

第一阶段可以上传：

1. 入口说明。
2. 规则说明。
3. 不含隐私正文的触发词表。
4. 同步水位记录。
5. 操作报告。
6. 脱敏后的示例模板。

## 4. 完成判断

GitHub PR 合并只表示“文件版本已更新”。

数字生命事项是否真正处理完成，仍以以下证据为准：

1. 本地总账已登记。
2. 必要规则已落地。
3. 必要时已生成 INBOX 回包或处理报告。
4. 未违反安全入库确认链。

## 5. 费用边界

普通文字文件、Markdown、规则和报告放入私有仓库，通常可使用 GitHub Free。

本阶段不使用：

1. GitHub Packages。
2. Codespaces。
3. 大量 Git LFS。
4. 高频 GitHub Actions。

因此本阶段原则上不触发额外 GitHub 费用。
