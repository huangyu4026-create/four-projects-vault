# 12｜手机录入口工程说明

## 定位

手机录入口是本身即美资料库的每日输入通道。

它解决的问题是：用户不需要复制粘贴长文字，而是在手机上打开一个网页，用手机键盘或语音输入，也可以顺手添加图片，提交后自动保存到 Mac 端工程文件夹。

录入口支持双工程分发：

| 目标工程 | 保存位置 |
|---|---|
| 本身即美 | `outputs/本身即美资料库` |
| 数字生命工程 | `outputs/数字生命资料库` |
| 红楼梦 | `outputs/红楼梦资料库` |

## 本身即美移动端工具区

手机录入口选择“本身即美”时，会显示本身即美专用工具区和总控区。

工具包：

- `场域理解包`：提交前进入本身即美、清风花园、人物历史和第二阶段语境。
- `前50集背景包`：按第二阶段前50集总指导判断真实生活事件、过程结果、干货情绪、院子气、季节气和生活气。
- `主题雷达`：判断题材优先级、雷达分、状态、缺口和下一步工程落点。
- `双面体系`：先定场，再开雷达。
- `分流规则包`：先按 28 号触发词总表判断意图，再按 41 号垂直通道规则做项目 × 阶段拆解。

总控区：

- `全局接管`：提交前确认总入口、工程状态快照、阶段封存表、前50集背景包、场域理解/主题雷达双面体系、28+41 分流规则和写入边界。
- `全局联动`：处理完成后复核主题状态、雷达排序、拍摄看板、单项目看板、项目归档、脚本、文案包、重要待定事项、待讨论区和排期表。

手机端工具包和总控区允许多选。点亮后，本次提交会同时写入正文前缀和触发词字段，并要求 AI 回答开头先确认已接收到哪些工具包或总控开关。

## 运行方式

```bash
python3 mobile-intake/server.py
```

服务启动后，手机和 Mac 在同一个 Wi-Fi 时，用手机浏览器打开：

```text
http://192.168.0.103:8787
```

具体地址以服务启动时终端输出为准。

## 保存位置

提交后的原文保存到：

```text
outputs/本身即美资料库/每日输入收件箱/
```

每日索引保存到：

```text
outputs/本身即美资料库/每日输入索引.md
```

每日原文汇总保存到：

```text
outputs/本身即美资料库/每日对话原文库/
outputs/本身即美资料库/每日对话原文库索引.md
```

图片材料保存到：

```text
outputs/本身即美资料库/图片材料收件箱/
outputs/本身即美资料库/图片材料索引.md
```

图片不是单独漂浮保存，而是绑定在同一条输入记录下面。比如“和田玉牌”的日记可以同时带图，原文文件里会出现“关联图片”，图片说明文件里也会反向记录“关联文字”。

每条输入默认都需要后续执行，任务保存到：

```text
outputs/本身即美资料库/任务执行台.md
```

分发索引生成命令：

```bash
python3 tools/process_mobile_inbox.py
```

分库待处理索引登记命令：

```bash
python3 tools/distribute_mobile_inbox.py
```

新版服务提交时会自动登记到分库待处理索引；该命令主要用于补处理历史收件箱。

重建每日原文汇总：

```bash
python3 tools/build_daily_input_archive.py
```

生成每日输入预处理报告：

```bash
python3 tools/preprocess_mobile_inbox.py
```

新版服务提交时会自动运行预处理脚本；该命令主要用于补处理历史收件箱或手动重建。

## 当前限制

1. Mac 必须开着。
2. 录入口服务必须运行。
3. 第一版要求手机和 Mac 在同一个 Wi-Fi。
4. 当前只自动进入收件箱，不自动完成深度分库回填。
5. 手机语音识别依赖手机系统键盘。
