# 000C_进入坐标查询_新窗口强制入口_候选正式头

生成时间：2026-06-21

性质：第二入口头部 / 坐标取材专用头 / 不替换旧入口

状态：候选正式头；不改旧库；不改旧 000；不改旧闭环包；只用于坐标法新窗口开头

## 一句话定案

红楼梦工程从此有两个头，后面的身子和尾巴共用。

旧头 / 聚拢头：

```text
进入聚拢查询
```

走旧语义 / 聚拢 / 闭环包路线。

新头 / 坐标头：

```text
进入坐标查询
```

或用户明确说：

```text
用坐标法查
坐标查询
坐标法查询
从坐标查
坐标取材
```

第一步必须走坐标取材门。

触发词里尽量不要写：

```text
进入红楼梦工程
进入红楼梦人工智能咨询工程
```

因为“红楼梦工程”会先触发旧 `000`，导致坐标题被旧闭环包吸走。

## 总铁律

坐标题不得先建旧闭环包。

坐标题第一动作是：

```text
取材门：坐标
recall_gate：coordinate
```

旧闭环包、聚拢库、材料池不是废掉，而是后面承接材料。

坐标头只替换“取材入口”，不替换整个工程身体。

## 继承铁律

坐标头不是野路子。

坐标头必须完整继承旧头的工程纪律：

```text
不允许先自由搜索 SQLite。
不允许先自由搜索全文。
不允许先凭印象回答。
不允许把“坐标命中”当作“可直接写答案”。
不允许把“距离最近”当作“已经裁判”。
不允许把“同原子段 / 同场 / 同事件”直接写成结论。
```

旧头的下列规则在坐标头中继续有效：

```text
人物先归一。
查询词不是答案。
命中只算召回。
坐标候选只算候选。
材料必须回原文。
材料必须入材料池。
没有材料池四态判定，不写最终答案。
没有精读材料词，不写最终答案。
没有写作前原文追证摘抄，不写最终答案。
```

坐标头只改变第一取材入口：

```text
旧头第一取材入口：闭环包 / 聚拢库取材单。
坐标头第一取材入口：坐标材料门。
```

## 双头结构

```text
旧头：进入聚拢查询
  -> 旧语义 / 聚拢闭环包取材
  -> 材料池
  -> 原文裁判
  -> 精读材料词
  -> 写作前原文追证
  -> 红楼解语

新头：进入坐标查询
  -> 坐标分流令牌
  -> 查询词路
  -> 人物归一强令牌
  -> 坐标材料门
  -> 全文词位库 / 变量投影库 / 字位桥表
  -> 距离 / 共场 / 交集
  -> 原子段与原文句材料候选
  -> 同一个材料池入口
  -> 原文裁判
  -> 精读材料词
  -> 写作前原文追证
  -> 红楼解语
```

## 坐标头强制流程

只要触发坐标头，必须按下列顺序：

```text
0. 确认坐标头触发。
1. 写入：
   取材门：坐标
   recall_gate：coordinate
2. 生成 Codex 查询词路。
3. 生成 Codex查询词、Codex强复合、Codex词角色、Codex查证顺序、Codex材料升级条件。
4. 人物 / 名字 / 称谓 / 别名题必须先过 70/72 全人物归一包。
5. 进入坐标材料门。
6. 有明确原文字词时，走全文词位材料索引库防漏。
7. 有人物、空间、季节、物象、事件时，走全量变量投影库定点。
8. 问最近 / 靠近 / 同场 / 同时 / 共同出现时，必须做距离、共场、交集。
9. 输出原子段、atom_code、segment_no、原文句、变量点、距离矩阵。
10. 送入同一个材料池入口。
11. 到材料池入口后，交还旧工程后半身。
```

## 禁止事项

坐标头触发后，禁止第一步做：

```text
python3 work/formal_honglou_closed_loop.py --question "用户原问题"
```

禁止第一步说：

```text
先建本题闭环包
先进入聚拢库取材单
先跑旧搜索词网络
先按旧语义召回
```

这些动作只能在坐标材料候选送入材料池以后，由旧工程后半身承接。

## 禁搬词表

以下旧头词语不能出现在坐标头第一动作中。

| 禁搬词 / 禁搬动作 | 为什么禁搬 | 坐标头替代 |
|---|---|---|
| 先建本题闭环包 | 会把坐标题吸回旧路线 | 先写入 `取材门：坐标` / `recall_gate：coordinate` |
| 先进入聚拢库取材单 | 会抢在坐标门之前 | 先进入坐标材料门 |
| 先跑旧搜索词网络 | 会变成语义补漏路线 | 先走全文词位库 + 坐标变量库 |
| 先按旧语义召回 | A/B 分流失败 | 坐标取材门直接取材 |
| 先验聚拢四件套 | 这是旧头前半段验收 | 坐标前半段验 `coordinate_preflight` 和坐标材料字段 |
| 用聚拢库取材证明坐标已进门 | 证明对象不对 | 用 `material_term_hits` / `material_coordinate_hits` / `atom_code` / `material_source_sentences` 证明 |

这些词不是废弃。

它们只能在坐标材料候选送入材料池以后，由旧工程后半身承接。

## 入口词规则

坐标头仍然尊重旧工程的成熟规则：

```text
人物先归一。
别名、称谓、异写先归到 character_key。
查询词不是答案。
命中不是答案。
材料必须回原文。
材料必须入材料池。
没有材料池裁判，不写最终答案。
```

但坐标头改变第一取材入口：

```text
旧头第一取材入口：聚拢闭环包。
坐标头第一取材入口：坐标材料门。
```

## 对象词提醒

对象词不要被单一路径锁死。

```text
全文词位负责防漏。
变量坐标负责定点。
变量未登记不等于原文没有。
精确变量也不能替代全文词位和原文裁判。
```

例如：

```text
花字：走全文词位 term_norm='花'
花物象：走变量 object=花
花相关扩展：由 AI 放在扩展词、背景锚点或补查方案中
```

最终靠人物、空间、季节、事件、距离 / 共场交叉裁判。

## 坐标头验收矩阵

坐标头成功与否，不用旧闭环包四件套验前半段。

必须用下表验收：

| 验收项 | 通过标准 |
|---|---|
| 坐标触发 | 出现 `取材门：坐标` 和 `recall_gate：coordinate`。 |
| 查询词路 | 出现 `Codex查询词`、`Codex强复合`、`Codex词角色`。 |
| 人物归一 | 人物题出现 `character_key` 和归一正名。 |
| 坐标预检 | `coordinate_preflight_status=通过`，或明确写出需补证令牌。 |
| 词位取材 | 有 `material_term_hits`，或说明本题没有明确字词入口。 |
| 变量取材 | 有 `material_coordinate_hits`，或说明本题没有变量入口。 |
| 距离 / 共场 | 最近、同场、同时、共同出现题必须有 `material_nearest_pairs` 或 `material_cooccurrence`。 |
| 原子段 | 有 `atom_code`、`segment_no`、`global_atom_order` 中的可用字段。 |
| 原文句 | 有 `material_source_sentences` 或等价原文句。 |
| 材料池入口 | 坐标候选已送入总证据池 / 材料池入口。 |
| 写作阻断 | 未完成材料池四态、精读材料词、写作前追证前，不写最终答案。 |

如果缺少坐标触发、查询词路、人物归一强令牌或坐标预检，状态只能写：

```text
坐标头阻断：未完成坐标取材入口。
```

如果已有坐标材料候选但尚未送入材料池入口，状态只能写：

```text
坐标取材已命中，待入材料池。
```

如果已送入材料池入口但后半身未完成，状态只能写：

```text
坐标材料已入池，待材料池四态 / 精读材料词 / 写作前追证。
```

## 状态词

坐标头只能这样报告状态：

```text
已触发坐标头 = 已识别坐标法，并生成取材门：坐标。
已进入坐标材料门 = 已通过坐标预检并调用坐标材料门。
已完成坐标取材 = 已提出原子段、原文句、变量点或词位点。
已送入材料池入口 = 坐标候选已进入总证据池 / 材料池入口。
可写最终答案 = 仍需旧工程后半身完成材料池四态、精读材料词、写作前原文追证。
```
