# 2026-06-06｜本身即美资料查询协议与纠错机制

## 形成原因

本协议来自一次实际错误：在查询“各阶段文案特点和代表正文”时，先依据当前输出层的小红书标题时间线做判断，误写为“缺少多数笔记完整正文”。后续回到原工程主轴后确认，`第一阶段连续轴/03｜第一阶段连续轴原文汇编.md` 中实际保存了大量正文文案、成品文案、用户口述要求、AI生成稿、发布版文案和文案规则。

错误本质不是单条结论错，而是查询路径错：没有先查主轴原文层，就对材料状态下了结论。

## 核心原则

以后凡是涉及“有没有原文、文案是什么、阶段怎样形成、某个类型从哪里来、某个判断依据是什么”的问题，一律不能只查当前输出文件。

必须按以下顺序完成查询：

1. 先查主轴原文层。
2. 再查阶段连续轴和主题时间线。
3. 再查原始材料暂存区。
4. 再查当前输出层。
5. 最后才做判断。

输出层只能作为索引和整理结果，不能作为唯一事实来源。

## 资料层级

| 层级 | 名称 | 作用 | 查询优先级 |
|---|---|---|---|
| L0 | 用户当前口述 | 最新修正、补丁、阶段解释、命名确认 | 最高 |
| L1 | 主轴原文层 | 原始正文、对话、成品文案、生成过程、真实讨论 | 必查 |
| L2 | 连续轴/时间线层 | 按时间组织后的阶段材料和主题材料 | 必查 |
| L3 | 原始材料暂存区 | 粘贴文本、小红书主页读取、网页读取、基础归档包 | 必查 |
| L4 | 阶段档案/产品线/战略研判 | 已沉淀的阶段结论、产品规则、战略判断 | 交叉验证 |
| L5 | 当前输出层 | 本轮生成的表、索引、摘要、阶段标签 | 只能辅助 |

## 固定查询路径

### 1. 查询文案正文

适用问题：

- 某阶段有没有正文？
- 某篇笔记原文是什么？
- 每个阶段的代表文案是什么？
- 某个文案风格从哪里来？

查询顺序：

| 步骤 | 查询动作 | 判断规则 |
|---|---|---|
| 1 | 查 `第一阶段连续轴/03｜第一阶段连续轴原文汇编.md` | 找标题、关键词、场景词、服装词、动作词、成品文案字段 |
| 2 | 查 `第一阶段连续轴/02｜第一阶段时间线（主题级）.md` | 确认该材料在连续轴中的主题位置 |
| 3 | 查 `99｜原始材料暂存区` | 找用户粘贴、主页读取、网页读取、基础归档 |
| 4 | 查阶段档案/产品线档案 | 判断它属于哪个阶段、哪条产品线、哪种战略 |
| 5 | 查当前 `outputs` | 对照已整理表是否遗漏或误标 |

禁止动作：

- 不能只看 `outputs/小红书时间线` 就判断“没有正文”。
- 不能只凭主页标题说“缺文案”。
- 不能把“没在当前表里找到”当成“工程里不存在”。

### 2. 查询阶段演化

适用问题：

- 第一阶段怎么分？
- 某个类型账号属于哪个阶段？
- 某个爆点和限流节点怎么发生？
- 第一阶段如何连接第二阶段？

查询顺序：

| 步骤 | 查询动作 | 目的 |
|---|---|---|
| 1 | 查用户最新口述补丁 | 用户口述具有最高修正权 |
| 2 | 查主页笔记时间线 | 确定发布顺序 |
| 3 | 查主轴原文 | 找当时的讨论和真实判断 |
| 4 | 查阶段档案 | 看已有阶段命名和归纳 |
| 5 | 查战略研判/执行台 | 看阶段如何转入后续执行 |

判断规则：

- 发布时间线和讨论时间线要分开。
- 文案生成日期不一定等于笔记发布日期。
- 一篇正文可以对应多个标签，但必须标明是“发布阶段”还是“生产讨论阶段”。

### 3. 查询产品线/愤怒潮湿

适用问题：

- 短裤文案怎么写？
- 愤怒潮湿如何进入本身即美？
- 产品推荐和普通文案有什么区别？

查询顺序：

| 步骤 | 查询动作 |
|---|---|
| 1 | 查 `产品线档案/愤怒潮湿｜短裤产品线档案.md` |
| 2 | 查 `小红书基础归档包/03｜愤怒潮湿｜短裤文案视觉协作与视频边界.md` |
| 3 | 查主页短裤/泳裤标题和点赞记录 |
| 4 | 查当前用户最新产品描述 |
| 5 | 再生成新文案或做阶段判断 |

规则：

- 产品文案不能脱离真实场景。
- 先查品牌规则，再写产品文案。
- “愤怒潮湿”与“本身即美”是关联账号/产品线关系，不能混写成普通第三方广告。

## 检索关键词设计

查询不能只搜标题，要同时搜四组关键词。

| 关键词组 | 示例 | 用途 |
|---|---|---|
| 标题词 | 古树修剪、黑皮嫩男、龙井茶园、愤怒潮湿、洗得干净 | 精确定位材料 |
| 场景词 | 花园、荷花、千岛湖、香港、纳雍、桐庐、龙井、院子、泳池 | 找同一阶段素材 |
| 服装词 | 白短裤、透明短裤、小马褂、西装、白衬衫、紧身裤、泳裤 | 找爆点和产品线 |
| 动作词 | 锄地、划船、桨板、跆拳、瑜伽、动物流、挑担、洗菜、拉车 | 找文案风格和身体结构 |
| 字段词 | 成品文案、发布版、用户原话、数据回填、核心一句、素材说明 | 找正文和规则 |

## 结论置信度

以后任何查询结论必须标注状态。

| 状态 | 含义 |
|---|---|
| 已确认 | 找到原文、标题、来源文件，阶段逻辑一致 |
| 候选确认 | 找到正文候选，但发布时间或标题还需对齐 |
| 只有标题 | 只有主页标题/赞数，没有找到正文 |
| 用户口述确认 | 用户已明确说明，文件中暂未完全找到 |
| 待核 | 目前证据不足，不能下确定结论 |

禁止使用模糊判断替代状态，例如“应该没有”“好像缺少”“可能不存在”。必须说明查了哪些层级、没查到的是哪一层。

## 反证机制

当出现“没找到”时，必须执行反证流程：

1. 至少换三类关键词再搜：标题词、场景词、动作词。
2. 至少查两个层级：主轴原文层和原始材料暂存区。
3. 如果只查了输出层，不能说没找到。
4. 如果主轴很大，先做关键词索引，再人工抽查上下文。
5. 输出时只能说“在已查范围内未找到”，不能说“没有”。

## 输出格式

以后查询类结果统一按这个格式输出：

```text
查询对象：
查询范围：
已查文件：
找到的证据：
未找到/待核部分：
结论状态：
下一步建议：
```

如果是阶段/文案归档，则增加：

```text
对应阶段：
对应标签：
原文状态：
是否代表性材料：
```
