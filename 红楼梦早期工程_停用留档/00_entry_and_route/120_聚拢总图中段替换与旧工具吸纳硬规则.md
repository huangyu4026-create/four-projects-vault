本次不再另起一套空的新工程，而是把旧工程最容易打架的中段替换为“聚拢总图法”：旧工具全部吸纳，旧中段指挥权清零。

```text
旧工具不丢。
旧前门不准抢跑。
旧中段不准并行。
聚拢总图是唯一中段指挥层。
所有旧工具产出的编号、候选、补点，必须回到聚拢总图再判断。
所有旧工具产出的命中，只能算召回，不能算写作放行。
```

## 查询词入口硬规则：单字根与疑义宽网

查询词不再允许只靠短语和临时联想词进入。

必须先做：

```text
题面词 -> 最小字根 -> 疑义宽网 -> 短语扩展 -> 图内收点 -> 编号过滤 -> 原文裁判
```

其中“最小字根”是硬要求。

```text
竹子 -> 竹
书信 / 推荐信 / 荐书 -> 书、信
宫灯 -> 灯
药方 -> 药、方
手帕 -> 帕
活鸡 -> 鸡
```

疑义词宁多勿少：

```text
对象不确定、叫法可能变化、原文可能有古典写法、物象可能被另称时，必须宽网进入。
20 个、30 个候选入口词可以接受。
找不到时先扩大，不要过早收窄。
```

确定专名不乱扩：

```text
林黛玉、贾宝玉、王熙凤这类明确人物，走人物归一和别名表。
不临时造一堆无证花名。
```

禁止把查询词工程做成“造花词工程”：

```text
先编出一堆竹板凳、竹衣服、竹扇子、竹画，
却没有先查 竹。
```

正确方式：

```text
单字宽收
-> 疑义处多词补网
-> 得到较多候选编号
-> 用人物、空间、动作、时间、物象类别、聚拢场过滤
-> 只把能回原文裁判的点送入材料池
```

这条规则的目的不是扩大噪音，而是防漏。

```text
单字负责不漏。
疑义宽网负责兜住变体。
短语负责收窄。
聚拢总图负责判断。
原文负责裁判。
```

## 穷尽法的硬位置

穷尽法不是旧搜索前门，也不是答案生成器。它是聚拢总图里的补点工具。

```text
题目里有小物件、小动作、小称谓、小词语、库可能没有的词
-> 用现成新编号入口门里的“全文穷尽收点”扫全文
-> 得到 segment_no / cluster_unit / event_id / chapter_no
-> 回聚拢总图找归属链、交集、场、域
-> 回原文裁判
```

它可以和人物归一、空间收点、两个或三个穷尽词组合使用。

## 最硬封锁规则

1. `库分级选库` 不再是运行前门。
2. `旧搜索词网络` 不得直接入池、不得直接出答案。
3. `旧八步主线` 不得在库外重新开一条并行主路。
4. `新编号入口门` 保留，但只作为聚拢总图内部收点工具。
5. `全文穷尽收点` 保留，但只负责补编号。
6. `聚拢总图` 是唯一中段指挥层。
7. 所有编号、候选、标签、节点、场、域，最终都必须回原文裁判。
8. 复杂题仍必须进入材料池、00M 原文追证摘抄和终稿后质量复核。
