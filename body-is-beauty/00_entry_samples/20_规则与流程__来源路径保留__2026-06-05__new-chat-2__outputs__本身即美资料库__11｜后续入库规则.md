# 05｜后续入库规则

## 新材料入库顺序

以后用户再给新材料，按以下顺序处理：

1. 保存原文，不改写。
2. 判断对象类型。
3. 分配分库。
4. 填通用字段。
5. 若是小红书笔记，加入笔记对象库。
6. 若有正文，加入文案库。
7. 若有用户口述判断，加入思考库。
8. 若有阶段/爆点/转型判断，加入论证库。
9. 若有新概念或新标签，加入定义库。
10. 回填来源库和查询状态。

新增补充：

所有从本身即美统一入口、手机录入口、网页总输入口或 Codex 对话推进中新形成的文件，都必须同时按 `2026-06-07｜本身即美文件归类一次性回填表` 做文件级归类。

## 入库对象类型

| 对象类型 | 进入主库 | 进入辅库 |
|---|---|---|
| 小红书笔记标题/主页数据 | 笔记对象库 | 数据反馈库、阶段库 |
| 小红书完整正文 | 文案库 | 笔记对象库、来源库 |
| 用户口述阶段解释 | 思考库 | 论证库、阶段库 |
| AI阶段总结 | 论证库 | 阶段库 |
| 产品文案/产品规则 | 产品库 | 文案库 |
| 爆点/限流数据 | 数据反馈库 | 论证库 |
| 标签/概念定义 | 定义库 | 全部分库引用 |
| 原始文件/导出文件 | 来源库 | 对应分库 |

## 入库状态

| 状态 | 含义 |
|---|---|
| 已确认 | 已找到来源，阶段和类型判断稳定 |
| 候选确认 | 有明确候选，但仍需对齐发布时间或原文 |
| 待核 | 证据不足，不能下定论 |
| 待回填 | 结构已建，但具体正文/来源未补 |
| 用户口述确认 | 用户明确说明，但文件证据未完全追到 |
| 已被替代 | 旧文件或旧判断已被修正版替代 |

## 正文处理规则

1. 正文原文不在总库里全文复制。
2. 总库只记录正文状态和来源路径。
3. 文案库可保存短摘录、文案风格和来源位置。
4. 如需完整原文，回到来源库对应文件。
5. 如果正文候选多于一个，保留多个候选，不强行合并。

## 分析规则

分析必须基于库对象，不直接凭记忆判断。

最低查询路径：

1. 查笔记对象库。
2. 查来源库。
3. 查文案库或思考库。
4. 查论证库。
5. 标明结论状态。

## 更新规则

每次新增材料，至少更新三处：

1. 对应分库。
2. 文件对象入库总表。
3. 来源库或笔记对象库。

如果新增材料形成稳定文件、HTML 看板、规则、项目归档、文案包或方案包，还要同步更新 `04｜分库索引.md`；必要时更新 `13｜每日输入分发索引.md`。

如果新增材料改变阶段判断，还要更新：

1. 阶段库。
2. 论证库。
3. 定义库。
